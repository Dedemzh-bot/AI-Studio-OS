### 角色私人宿舍 - 优秀案例

#### 💡 【核心经验总结】 [读取优先级：Highest]
*(提炼1~3条正向设计原则，告诉未来AI必须复用什么架构思维。)*
- **【核心范式】**：**“对于玩法系统设计，条件-流转-数据-表现-异常”五段式结构**。这是本设计的灵魂。每个玩法子系统都严格遵循此范式，确保了设计的完整性、可读性和可落地性，是未来所有玩法系统设计的标准模板。
- **【核心范式】**：**“前置条件”是系统安全的防火墙，该条件是触发在玩法系统当中，作为项目概述和设计目的不需要**。每个子系统都明确列出了进入前的强依赖（如：拥有角色、购买DLC、好感度达标、次数未耗尽）。这从根本上杜绝了“状态错乱”和“非法访问”的bug，是防御性编程的顶层设计。
- **【核心范式】**：**“边界与异常兜底”是系统的最后一道防线**。每个子系统都预设了加载失败、数据损坏、网络中断、次数耗尽、超时等常见异常，并给出了明确的降级或恢复策略。这保证了系统在任何极端情况下都能优雅地处理错误，而非直接崩溃。

#### 🦴 【优秀脱水骨架】 [读取优先级：Low]
*(提取原文档中优秀的纯逻辑骨架，作为成功落地的参考。)*
- **角色私人宿舍系统**
  - **【通用前置与流转】**
    - 入口：主界面 -> 角色选择 -> 加载专属房间
    - 核心交互：点击房间内热点 -> 触发互动动画 -> 返回AI状态
    - 退出：返回角色选择界面或主界面
  - **【数据与表现反馈】**
    - 数据层：读取 `room_config`, `current_outfit_id`, `hotspot_status` 等配置
    - 表现层：房间淡入动画、角色“欢迎回家”动作、视角控制UI、退出告别动画
  - **【异常与兜底机制】**
    - 网络中断：弹窗提示，返回主界面
    - 模型加载超时：显示进度条，超时后降级为低精度模型
    - 配置数据损坏：恢复默认布局，记录错误日志

- **互动热点系统**
  - **【通用前置与流转】**
    - 条件：角色已加载、热点未锁定、非拍照模式
    - 流程：靠近高亮 -> 点击触发 -> 读取动画ID -> 打断AI行为 -> 播放互动 -> 返回AI状态
  - **【数据与表现反馈】**
    - 数据：更新 `hotspot_status`, `affection_exp`, `daily_interaction_count`
    - 表现：热点光晕图标、UI隐藏、强制运镜、好感度气泡提示
  - **【异常与兜底机制】**
    - 玩家跳过动画：立即结束，奖励照常发放
    - 次数耗尽：热点变灰，文字提示
    - 好感度不足：显示锁图标，文字提示
    - 动画文件缺失：播放默认动画，记录错误日志

- **小游戏系统**
  - **【通用前置与流转】**
    - 条件：好感度达标、非冷却期
    - 流程：点击入口 -> 加载游戏场景 -> 进行游戏 -> 判定结果 -> 发放奖励 -> 返回房间
  - **【数据与表现反馈】**
    - 数据：更新 `daily_minigame_count`, `affection_exp`, `minigame_achievements`
    - 表现：3D入口物体、淡入转场、游戏内UI、结果弹窗、角色“等你回来”动作
  - **【异常与兜底机制】**
    - 次数耗尽：入口变灰，文字提示
    - 断线：游戏暂停，限时重连
    - 加载超时：提示失败，返回房间
    - 得分异常：强制判定失败，记录错误日志

- **好感度系统**
  - **【通用前置与流转】**
    - 条件：房间已加载，独立于主游戏
    - 流程：获得经验 -> 累计 -> 达到阈值 -> 升级 -> 解锁内容
  - **【数据与表现反馈】**
    - 数据：更新 `affection_exp`, `affection_level`, `unlocked_content`
    - 表现：经验气泡、升级动画、解锁提示、等级UI、联动奖励弹窗
  - **【异常与兜底机制】**
    - 经验溢出：停止累积，显示“已满”提示
    - 阈值数据错误：锁定等级为0，记录错误日志
    - 联动奖励发放失败：提示联系客服，记录错误日志

- **剧情解锁系统**
  - **【通用前置与流转】**
    - 条件：好感度达标或完成成就
    - 流程：解锁提示 -> 进入播放界面 -> 加载剧情 -> 播放 -> 标记已观看
  - **【数据与表现反馈】**
    - 数据：更新 `unlocked_story_segments`, `story_segment_status`, `story_choices`
    - 表现：手机消息提示、全屏UI、剧情结束提示、档案系统状态
  - **【异常与兜底机制】**
    - 文件加载失败：提示重试，返回房间
    - 中途退出：自动保存进度
    - 选项数据异常：强制跳过，使用默认分支

- **纪念品系统**
  - **【通用前置与流转】**
    - 条件：达成成就或好感度达标
    - 流程：判定解锁 -> 检查碎片 -> 合成/显示进度 -> 自动摆放
  - **【数据与表现反馈】**
    - 数据：更新 `souvenir_fragment_count`, `unlocked_souvenirs`, `room_decorations`
    - 表现：解锁提示、碎片进度条、合成动画、3D模型展示
  - **【异常与兜底机制】**
    - 合成失败：提示重试，不扣除碎片
    - 摆放位置被占：自动调整到下一位置
    - ID不存在：记录错误日志
    - 好感度解锁：跳过碎片检查

- **AI日常行为系统**
  - **【通用前置与流转】**
    - 条件：房间已加载，非互动/小游戏状态
    - 流程：状态机控制 -> 执行行为 -> 随机切换 -> 可被玩家打断
  - **【数据与表现反馈】**
    - 数据：更新 `ai_behavior_state`, `ai_behavior_timer`, `random_voice_timer`
    - 表现：对应动画、移动路径、对话气泡、小动作
  - **【异常与兜底机制】**
    - 动画缺失：保持“空闲”状态
    - 路径被阻挡：自动绕行或等待
    - 长时间无互动：触发“呼唤玩家”行为
    - 状态机数据错误：强制重置为“空闲”

- **拍照模式**
  - **【通用前置与流转】**
    - 条件：房间已加载，随时可开启
    - 流程：点击相机 -> 调整视角/滤镜/姿势 -> 拍照 -> 保存 -> 退出
  - **【数据与表现反馈】**
    - 数据：更新 `photo_mode_active`, `character_pose`, `character_expression`
    - 表现：专属UI、网格辅助线、实时滤镜预览、姿势切换动画、音效闪光灯
  - **【异常与兜底机制】**
    - 点击热点：自动退出拍照模式并触发互动
    - 保存失败：提示清理存储空间
    - 姿势/表情数据不存在：使用默认值
    - 断线：自动退出拍照模式，返回房间
