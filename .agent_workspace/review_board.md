# 冰霜新星技能数据结构验收表

## 字段说明

| 字段名 | 含义 | 取值范围/格式 | 必填 | 防漏洞措施 |
|--------|------|---------------|------|------------|
| skill_id | 技能唯一标识 | 必须为 `sk_ice_frost_nova`，遵循全局命名规范 | 是 | 固定值防止ID篡改 |
| skill_name | 技能显示名称 | 只能是 "冰霜新星" 或 "FrostNova" | 是 | 枚举限制，防止乱填 |
| description | 技能描述文本 | 字符串，最长200字符 | 是 | 长度限制防止注入 |
| base_damage | 初始基础伤害值 | 整数，0~100 | 是 | 上限100，遵守数值平衡规范，防止前期体验破坏 |
| freeze_duration | 冻结持续时间 | 数字，0~10秒 | 是 | 上限10秒，防止无限控制 |
| radius | 技能影响半径 | 数字，0~500单位 | 是 | 上限500，防止全屏覆盖 |
| trigger_source | 触发来源对象 | 包含 source_type 和 source_id | 是 | 强制追踪来源，防止地图边缘爆炸等异常触发 |
| trigger_source.source_type | 触发来源类型 | 只能是 "character"、"item"、"environment"、"system" | 是 | 枚举限制，防止伪造来源 |
| trigger_source.source_id | 触发来源标识 | 字母数字下划线组成，如 "Mage_012" | 是 | 正则校验，防止特殊字符注入 |
| causality_trace | 因果追踪日志数组 | 包含事件、时间戳、来源的数组 | 否 | 记录完整事件链，用于审计和漏洞排查 |

## 防漏洞设计要点

1. **触发源追踪**：每个技能实例必须记录触发来源（角色/物品/环境/系统），防止无来源的异常触发导致无限刷怪或资源溢出。

2. **因果日志**：可选但推荐的追踪数组，记录从技能激活到效果执行的每个事件，便于回溯异常行为。

3. **数值上限**：基础伤害上限100、冻结时间上限10秒、半径上限500，防止数值溢出破坏平衡。

4. **命名规范**：skill_id 固定为 `sk_ice_frost_nova`，防止ID混淆或注入。

5. **类型严格**：所有字段类型和枚举值严格限定，禁止额外字段（additionalProperties: false）。
