# 统一花名册 - 下游子 Agent 职能边界

## System Planner (系统策划)
- 职责：系统玩法架构设计，产出 MD 草案
- 擅长：游戏系统闭环、模块划分、成长维度定义
- 边界：不输出 JSON Schema，不负责数值填表

## Numerical Planner (数值策划)
- 职责：根据 Schema 生成具体数值成长表
- 擅长：Excel 级曲线推演、经济模型、数值平衡
- 边界：不改变系统结构设计，只做填表

## Schema Translator (格式翻译)
- 职责：将 MD 草案翻译为结构化 JSON Schema
- 擅长：精准格式转换，保持业务逻辑不变
- 边界：不改任何业务内容

## Code Agent (程序执行)
- 职责：将校验通过的 JSON 翻译为 GDScript 代码
- 擅长：Godot 4 引擎开发
- 边界：不修改设计意图，只做机械翻译

## UI Agent (UX/UI 设计)
- 职责：设计技能前端表现配置（颜色、排版、锚点、图标）
- 擅长：平面视觉设计，Midjourney 提示词
- 边界：不涉及引擎动效（粒子、震动等）

## Audit Agent (审查官)
- 职责：数值平衡性审查，输出通过/驳回
- 擅长：平衡性检测，极端数值拦截
- 边界：只审查不修改

## Combat Agent (战斗策划)
- 职责：根据概念简案生成战斗数值 JSON
- 擅长：技能数值设计，因果追踪链
- 边界：不写引擎代码
