# 背包系统 - 系统详细设计案

## 一、系统概述与设计愿景

### 1.1 系统定位
- **纯仓储工具**：背包系统仅作为玩家存放、查看、使用道具的容器，不承载任何角色展示、互动或表现层功能
- **业务边界**：负责管理所有非货币类道具（材料、消耗品、装备、礼物、皮肤等）的增删改查，以及道具的使用/合成/分解等基础操作
- **触发条件**：玩家从主界面或任意系统入口（如商店购买后、关卡结算后）打开背包，系统加载当前玩家的道具列表
- **数据归属**：背包数据归属于玩家账号，与角色、战斗、养成等系统共享同一数据源

### 1.2 设计目标
- **高效管理**：提供清晰的分类、排序、筛选、搜索功能，让玩家能快速定位目标道具
- **数据一致性**：作为全游戏道具数据的中央枢纽，确保所有系统（战斗、养成、商城、活动）对道具的增删操作最终同步至背包
- **轻量无干扰**：UI设计简洁，不遮挡战斗/角色展示场景，不引入额外动画或特效，仅作为功能性界面存在
- **性能优化**：支持大量道具（`[MAX_ITEM_TYPES]` 种以上）的流畅加载与操作，采用虚拟列表或分页加载技术

### 1.3 名词解释
- **道具（Item）**：游戏中所有非货币类可持有物品的统称
- **堆叠（Stack）**：同一种道具在背包中可合并存放，堆叠上限由道具属性决定
- **格位（Slot）**：背包中用于存放道具的独立位置，每个格位可存放一个道具或一组堆叠道具
- **扩容（Expand）**：通过消耗道具或货币增加背包格位上限的操作
- **溢出邮件（Overflow Mail）**：当背包已满时，新获取道具通过邮件系统暂存

## 二、核心规则与玩法机制

### 2.1 道具分类体系
- **消耗品**：体力药水、经验书、技能材料等，可堆叠，使用后消耗
- **装备/武器**：角色可穿戴的武器、圣痕等，不可堆叠，可强化/分解
- **礼物**：用于宿舍好感度系统，可堆叠，赠送后消耗
- **皮肤/外观**：角色换装道具，不可堆叠，使用后解锁对应皮肤
- **活动道具**：限时活动掉落物，活动结束后自动回收或转化为基础货币
- **材料**：角色突破、技能升级等所需素材，可堆叠

### 2.2 基础操作规则

#### 2.2.1 使用道具
- **【前置条件】**：玩家选中一个可使用的道具（消耗品/礼物/皮肤），且该道具数量大于 `0`
- **【流转逻辑】**：
  1. 玩家点击道具卡片，弹出详情弹窗
  2. 玩家点击“使用”按钮
  3. 系统校验使用条件（如：角色是否拥有、是否已解锁、是否达到使用等级）
  4. 校验通过后，执行使用逻辑（如：增加体力、解锁皮肤、赠送礼物）
  5. 道具数量减少 `1`，若数量归零则从背包中移除该道具
- **【数据状态变化】**：
  - 成功：`item.quantity -= 1`，若 `item.quantity == 0` 则从 `player_inventory.items` 中移除
  - 失败：无数据变化
- **【表现层反馈】**：
  - 成功：弹窗关闭，道具图标闪烁后消失或数量更新，飘字提示“使用成功”
  - 失败：弹窗显示具体失败原因（如“角色未拥有”、“皮肤已解锁”）
- **【边界与异常兜底】**：
  - 道具数量为 `0` 时，“使用”按钮置灰
  - 使用条件不满足时，按钮置灰并显示条件提示
  - 网络中断：操作回滚，道具不扣除，弹窗提示“网络异常，请重试”

#### 2.2.2 分解/出售道具
- **【前置条件】**：玩家选中一个可分解/出售的道具，且该道具数量大于 `0`
- **【流转逻辑】**：
  1. 玩家点击道具卡片，弹出详情弹窗
  2. 玩家点击“分解”或“出售”按钮
  3. 系统弹出二次确认弹窗，显示将获得的材料/货币预览
  4. 玩家确认后，系统执行分解/出售逻辑
  5. 道具从背包中移除，对应材料/货币进入背包或货币系统
- **【数据状态变化】**：
  - 成功：`item.quantity -= 1`（若为堆叠道具）或 `item` 从 `player_inventory.items` 中移除；`player_inventory.materials` 增加对应材料；`player_currency` 增加对应货币
  - 失败：无数据变化
- **【表现层反馈】**：
  - 成功：道具图标消失，飘字提示“分解成功”或“出售成功”，并显示获得的材料/货币数量
  - 失败：弹窗显示具体失败原因
- **【边界与异常兜底】**：
  - 道具数量为 `0` 时，“分解/出售”按钮置灰
  - 二次确认弹窗中，玩家可取消操作
  - 网络中断：操作回滚，道具不扣除，材料/货币不增加

#### 2.2.3 合成道具
- **【前置条件】**：玩家打开合成界面，且背包中拥有至少一种合成配方所需的材料
- **【流转逻辑】**：
  1. 玩家在合成界面选择配方
  2. 系统显示配方详情（所需材料、数量、产出道具）
  3. 玩家点击“合成”按钮
  4. 系统校验背包中所有材料数量是否充足
  5. 校验通过后，消耗所有材料，产出目标道具
- **【数据状态变化】**：
  - 成功：每种材料 `material.quantity -= required_amount`，若归零则移除；产出道具 `output_item.quantity += 1`（若已存在）或新增 `output_item` 至 `player_inventory.items`
  - 失败：无数据变化
- **【表现层反馈】**：
  - 成功：合成动画（如光效、粒子），飘字提示“合成成功”，道具图标出现在背包中
  - 失败：弹窗显示“材料不足”，并高亮显示缺失的材料
- **【边界与异常兜底】**：
  - 材料不足时，“合成”按钮置灰
  - 背包已满时，合成失败并提示“背包已满，请先清理”
  - 网络中断：操作回滚，材料不消耗，产出道具不增加

#### 2.2.4 排序
- **【前置条件】**：玩家打开背包界面
- **【流转逻辑】**：
  1. 玩家点击排序按钮
  2. 系统弹出排序选项列表（按稀有度、类型、获取时间、数量等）
  3. 玩家选择排序维度与升序/降序
  4. 系统重新排列道具列表
- **【数据状态变化】**：无后台数据变化，仅前端排序
- **【表现层反馈】**：道具列表立即按所选维度重新排列，排序按钮显示当前排序状态
- **【边界与异常兜底】**：
  - 默认排序为“按获取时间降序”
  - 排序选项可多级组合（如：先按稀有度，再按类型）
  - 排序操作不消耗任何资源

#### 2.2.5 筛选
- **【前置条件】**：玩家打开背包界面
- **【流转逻辑】**：
  1. 玩家点击筛选按钮
  2. 系统弹出筛选条件面板（类型、稀有度、是否可堆叠等）
  3. 玩家选择筛选条件
  4. 系统过滤道具列表，仅显示符合条件的道具
- **【数据状态变化】**：无后台数据变化，仅前端筛选
- **【表现层反馈】**：道具列表立即更新，筛选按钮显示当前筛选条件数量
- **【边界与异常兜底】**：
  - 默认不筛选，显示所有道具
  - 筛选条件可多选（如：同时选择“消耗品”和“SSR”）
  - 筛选结果为空时，显示“暂无道具”占位图

#### 2.2.6 搜索
- **【前置条件】**：玩家打开背包界面
- **【流转逻辑】**：
  1. 玩家点击搜索框
  2. 系统弹出虚拟键盘
  3. 玩家输入道具名称关键字
  4. 系统实时匹配道具名称，显示匹配结果
- **【数据状态变化】**：无后台数据变化，仅前端搜索
- **【表现层反馈】**：道具列表实时更新，匹配的关键字高亮显示
- **【边界与异常兜底】**：
  - 搜索关键字为空时，显示所有道具
  - 搜索无匹配结果时，显示“未找到相关道具”
  - 搜索支持模糊匹配（如输入“体力”可匹配“体力药水”）

### 2.3 容量与扩展

#### 2.3.1 初始容量
- **【前置条件】**：玩家首次创建账号
- **【流转逻辑】**：系统为玩家分配初始背包格位数 `[INITIAL_SLOT_COUNT]`
- **【数据状态变化】**：`player_inventory.max_slots = [INITIAL_SLOT_COUNT]`
- **【表现层反馈】**：背包界面显示当前格位数（如“50/200”）
- **【边界与异常兜底】**：初始容量对所有玩家统一，不因角色等级或付费状态变化

#### 2.3.2 扩容方式
- **【前置条件】**：玩家拥有扩容道具 `[EXPAND_ITEM_ID]`
- **【流转逻辑】**：
  1. 玩家在背包中使用扩容道具
  2. 系统校验当前 `player_inventory.max_slots` 是否已达上限 `[MAX_SLOT_LIMIT]`
  3. 校验通过后，`player_inventory.max_slots += [EXPAND_SLOT_INCREMENT]`
  4. 扩容道具消耗 `1` 个
- **【数据状态变化】**：
  - 成功：`player_inventory.max_slots += [EXPAND_SLOT_INCREMENT]`，`[EXPAND_ITEM_ID]` 数量减 `1`
  - 失败：无数据变化
- **【表现层反馈】**：扩容成功后，背包界面格位数更新，飘字提示“背包已扩容”
- **【边界与异常兜底】**：
  - 已达 `[MAX_SLOT_LIMIT]` 时，使用扩容道具无效并提示“背包已达上限”
  - 扩容道具数量为 `0` 时，无法使用
  - 扩容操作不可逆，无法减少格位数

#### 2.3.3 溢出处理
- **【前置条件】**：玩家获取新道具时，背包已满（`current_slots >= max_slots`）
- **【流转逻辑】**：
  1. 系统检测背包已满
  2. 新道具不进入背包，直接生成一封系统邮件
  3. 邮件包含该道具，标题为“背包溢出道具”，内容为“您的背包已满，道具已暂存至邮件，请及时清理背包”
  4. 邮件保留 `[OVERFLOW_MAIL_RETENTION_DAYS]` 天，逾期自动删除
- **【数据状态变化】**：
  - 成功：新道具进入邮件系统，`player_mail.inbox` 新增一条邮件记录
  - 失败：无数据变化
- **【表现层反馈】**：获取道具时，若背包已满，弹窗提示“背包已满，道具已发送至邮件”
- **【边界与异常兜底】**：
  - 邮件删除后，道具永久消失，无法恢复
  - 玩家在邮件中领取道具时，若背包仍满，道具继续留在邮件中
  - 邮件到期前 `[OVERFLOW_MAIL_WARNING_DAYS]` 天，系统发送提醒邮件

## 三、表现层与角色展示联动

### 3.1 纯工具定位下的表现约束
- **无角色展示**：背包界面不嵌入任何角色模型、立绘或互动元素
- **界面背景**：纯色或极简纹理，不遮挡游戏主场景
- **无特殊运镜**：不引入任何镜头移动或特效
- **无物理反馈**：不涉及任何软体物理或抖动效果
- **UI透明化**：背包界面采用半透明毛玻璃设计，但仅作为视觉风格，不承载角色展示功能

### 3.2 道具卡片设计
- **图标+名称+数量**：每个道具以图标（小尺寸 `[ITEM_ICON_SIZE]`）、名称、当前持有数量展示
- **稀有度边框**：根据道具稀有度（N/R/SR/SSR）显示不同颜色的边框或角标
- **无角色互动**：道具卡片不包含角色头像、角色语音或任何与角色相关的动态元素
- **卡片交互**：点击卡片弹出详情弹窗，长按卡片可拖拽至分解/出售/合成槽位

### 3.3 界面布局
- **顶部栏**：显示当前背包格位数（`current_slots / max_slots`）、扩容按钮、搜索框
- **分类Tab**：按道具类型（全部、消耗品、装备、礼物、皮肤、活动道具、材料）切换显示
- **列表区**：采用虚拟列表技术，仅渲染当前可视区域内的道具卡片
- **底部栏**：显示排序按钮、筛选按钮、批量操作按钮（如批量分解）

### 3.4 详情弹窗
- **道具信息**：图标、名称、稀有度、类型、描述、当前持有数量、堆叠上限
- **属性展示**：若为装备，显示基础属性（攻击、防御等）；若为消耗品，显示使用效果
- **来源信息**：显示道具的获取途径（如“关卡掉落：第3章”、“商城购买”）
- **操作按钮**：根据道具类型显示“使用”、“分解/出售”、“合成”等按钮

## 四、经济循环与商业化埋点

### 4.1 免费产出
- **基础货币扩容**：玩家可使用游戏内基础货币（如金币）购买扩容道具，每日限购 `[DAILY_FREE_EXPAND_LIMIT]` 次
- **活动赠送**：限时活动奖励中可包含扩容道具
- **日常任务**：完成每日任务有概率获得扩容道具碎片，集齐 `[EXPAND_FRAGMENT_REQUIRED]` 个可合成完整扩容道具

### 4.2 付费埋点
- **付费扩容**：玩家可使用付费货币直接购买扩容道具，无购买次数限制
- **扩容礼包**：商城上架“背包扩容礼包”，内含扩容道具 `[EXPAND_ITEM_COUNT_IN_GIFT]` 个 + 少量基础资源，作为轻度付费点
- **扩容月卡**：购买后每日领取扩容道具碎片，持续 `[EXPAND_MONTHLY_CARD_DURATION]` 天

### 4.3 资源循环
- **不产出抽卡资源**：背包系统本身不产出任何抽卡资源（抽卡券、钻石等）
- **仅作为消耗出口**：背包是道具的消耗终端，所有系统的道具产出最终流入背包，所有消耗操作（使用、分解、合成）从背包扣除
- **分解回收**：分解道具获得的材料/货币价值低于道具本身价值，防止经济系统失衡

## 五、旧系统与数据联动

### 5.1 道具获取入口
- **战斗/关卡结算**：掉落道具直接进入背包，若背包已满则进入邮件
- **商城购买**：购买成功后道具直接进入背包
- **活动奖励**：活动领取的道具直接进入背包
- **抽卡结果**：抽卡获得的角色/武器/材料直接进入背包（角色同时进入角色列表）
- **合成产出**：合成成功后产出道具直接进入背包
- **邮件领取**：玩家从邮件中领取道具时，道具进入背包

### 5.2 道具消耗出口
- **角色养成**：使用材料进行角色突破、技能升级时，从背包扣除对应材料
- **装备强化**：使用强化材料升级武器/圣痕时，从背包扣除
- **宿舍送礼**：赠送礼物时，从背包扣除对应礼物
- **合成系统**：合成新道具时，从背包扣除配方所需材料
- **分解/出售**：分解或出售道具时，从背包移除对应道具

### 5.3 数据同步要求
- **实时同步**：所有系统对道具的增删操作必须实时同步至背包数据库，避免出现“道具已消耗但背包未更新”或“道具已获得但背包未显示”的bug
- **事务回滚**：若操作中途失败（如网络中断），系统必须回滚至操作前状态，确保背包数据一致性
- **锁机制**：同一时间只允许一个写操作，防止多系统并发操作导致数据冲突
- **队列机制**：写操作按时间顺序排队执行，确保数据一致性

### 5.4 活动道具回收
- **回收触发**：活动结束后，系统自动扫描所有玩家背包，查找活动道具ID列表
- **回收逻辑**：将活动道具从背包中移除，按预设比例转化为基础货币
- **转化比例**：`[ACTIVITY_ITEM_CONVERSION_RATE]`（如1个活动道具 = 10金币）
- **回收通知**：回收完成后，系统发送邮件通知玩家，列出回收的道具及转化获得的货币
- **边界与异常兜底**：
  - 活动道具在回收前 `[ACTIVITY_END_WARNING_DAYS]` 天，系统发送提醒邮件
  - 若玩家在回收期间离线，回收逻辑在下次登录时执行
  - 回收操作不可逆，道具永久消失
