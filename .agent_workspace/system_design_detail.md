# 一、系统概述与设计愿景

【角色私人宿舍】系统是一个**买断制解锁**的、为每位女性角色量身定制的**专属私密空间**。其核心设计愿景是：将角色从“战斗工具”升维为“可陪伴的恋人”。通过高度定制化的房间装修、专属好感度对话链、特殊触摸反馈及私密事件，深度强化玩家与单一角色的情感羁绊（Master-Love），完美契合《Project T》中“极致表现层”与“ML向情感羁绊”的核心支柱。该系统是游戏内除战斗外，最重要的角色魅力展示与情感付费转化场景。

# 二、核心规则与玩法机制

## 2.1 解锁条件

- **【前置条件】**：玩家需在商城中**一次性付费购买**“私人宿舍”核心模块（定价见第五节）。
- **【流转逻辑】**：
    1. 玩家在商城界面点击“私人宿舍”商品，弹出购买确认弹窗。
    2. 玩家确认支付（扣除对应货币）。
    3. 系统校验支付成功，将 `PlayerDormitoryData.isUnlocked` 字段设为 `true`。
    4. 系统遍历玩家当前拥有的所有女性角色列表，为每个角色生成初始化的 `PlayerDormitoryData.roomData[roleId]` 记录。
    5. 弹出“宿舍已解锁”全屏庆祝动画，并引导玩家进入任意角色的房间。
- **【数据状态变化】**：
    - 成功：`PlayerDormitoryData.isUnlocked = true`；`PlayerDormitoryData.roomData` 新增所有已拥有角色的默认房间数据。
    - 失败（货币不足）：无数据变化。
- **【表现层反馈】**：
    - 成功：播放解锁动画，显示“恭喜解锁【私人宿舍】！”文字提示。
    - 失败：弹出“货币不足”提示窗，并引导至充值页面。
- **【边界与异常兜底】**：
    - 若玩家在购买过程中断线，下次登录时，系统需重新校验订单状态。若支付成功但数据未写入，则补写数据并弹出“宿舍已解锁”提示。
    - 若玩家购买后获得新角色，系统在角色入队时自动执行步骤4的逻辑，无需玩家再次操作。

## 2.2 玩法循环

- **【前置条件】**：`PlayerDormitoryData.isUnlocked == true`；玩家拥有至少一名女性角色。
- **【流转逻辑】**：
    1. **进入**：玩家在主界面点击“宿舍”入口，进入角色选择界面。
    2. **选择角色**：玩家从已拥有角色列表中选择一名角色，点击“进入房间”。
    3. **加载房间**：系统根据 `PlayerDormitoryData.roomData[roleId]` 加载该角色的专属房间场景（包含家具摆放、灯光设置等自定义数据）。
    4. **核心互动**：
        - **自由视角**：玩家通过拖拽/滑动屏幕旋转视角，双指缩放。视角范围限制：水平360度，垂直-30度至+60度，最近缩放距离为角色模型前方0.5米。
        - **触摸反馈**：玩家点击角色身体不同部位，触发对应反馈。
        - **好感度对话**：玩家点击对话按钮，弹出好感度等级对应的对话选项列表。
        - **换装展示**：角色自动穿着玩家当前为该角色装备的服装/皮肤，并播放待机动作。
    5. **退出**：玩家点击返回按钮，退出房间，回到角色选择界面。
- **【数据状态变化】**：
    - 每次有效触摸互动：`PlayerDormitoryData.roomData[roleId].dailyInteractionCount += 1`；`PlayerDormitoryData.roomData[roleId].affectionExp += 10`。
    - 每次完成好感度对话：`PlayerDormitoryData.roomData[roleId].affectionExp += 50`。
    - 好感度升级时：更新 `PlayerDormitoryData.roomData[roleId].affectionLevel`，并触发解锁内容。
- **【表现层反馈】**：
    - 进入房间：播放专属运镜动画（从角色脚部缓慢上移至面部），随后UI渐入，显示极简操作提示。
    - 触摸反馈：播放对应语音、表情动画、Jiggle Physics。
    - 好感度对话：弹出对话气泡，显示选项文本，选择后播放剧情片段。
    - 换装展示：角色模型实时更新为当前装备的服装。
- **【边界与异常兜底】**：
    - 若玩家在互动过程中断线，当前互动数据（如触摸次数、好感度经验）在下次登录时，以服务器最后保存的数据为准，本地缓存仅作为临时展示。
    - 若玩家在对话选择过程中断线，该次对话不计入完成次数，不消耗每日对话次数上限。

## 2.3 重置与状态

- **【前置条件】**：每日服务器时间到达00:00。
- **【流转逻辑】**：
    1. 服务器定时任务触发，遍历所有玩家的 `PlayerDormitoryData`。
    2. 对于每个角色，将 `PlayerDormitoryData.roomData[roleId].dailyInteractionCount` 重置为0。
    3. 对于每个角色，将 `PlayerDormitoryData.roomData[roleId].dailyDialogueCount` 重置为0。
- **【数据状态变化】**：
    - `PlayerDormitoryData.roomData[roleId].dailyInteractionCount = 0`
    - `PlayerDormitoryData.roomData[roleId].dailyDialogueCount = 0`
- **【表现层反馈】**：无直接表现层反馈。玩家下次进入宿舍时，UI上的每日次数显示会更新为0。
- **【边界与异常兜底】**：
    - 若玩家在重置时刻（00:00）正处于宿舍互动中，当前正在进行的互动正常完成，但完成后立即触发重置逻辑，后续互动计入新一天。
    - 重置逻辑必须保证原子性，避免因服务器压力导致部分角色重置成功、部分失败的情况。

# 三、表现层与角色展示联动 (核心强制项)

## 3.1 最大化角色展示

- **【前置条件】**：玩家进入任意角色的专属房间。
- **【流转逻辑】**：
    1. 加载房间场景后，立即触发“专属运镜”动画：摄像机从角色脚部（地面高度）开始，以每秒0.3米的速度缓慢上移，历时3秒，最终定格在角色面部特写（摄像机位于角色前方1.5米，高度与角色眼睛平齐）。
    2. 运镜动画播放完毕后，自动切换至“玩家可控视角”模式。
    3. 在可控视角模式下，所有UI按钮（返回、对话、设置等）在玩家无操作3秒后自动半透明化（透明度降至20%），并在玩家触摸屏幕任意位置时立即恢复至100%不透明度。
    4. 角色模型渲染优先级最高，确保在任何视角下，角色模型不被任何UI元素遮挡。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 运镜动画：屏幕上下出现黑边（模拟电影宽屏效果），底部显示角色名字与当前好感度等级（白色半透明字体，持续5秒后消失）。
    - UI极简化：按钮半透明化时，伴随0.5秒的淡出动画。
    - 动态环境：房间内主光源（如窗外的月光）会随角色头部转动产生0.5秒延迟的跟随效果；角色身体移动时，台灯光源投射在墙上的阴影会实时更新。
- **【边界与异常兜底】**：
    - 若玩家在运镜动画播放过程中点击屏幕，立即中断运镜，直接切换至可控视角。
    - 若玩家设备性能不足，导致动态光影计算卡顿，系统自动降级为静态光影（预烘焙光照贴图），并在UI上显示“性能模式已启用”提示（仅首次触发时显示）。

## 3.2 擦边互动与物理反馈

- **【前置条件】**：玩家处于可控视角模式，且手指点击到角色身体碰撞体。
- **【流转逻辑】**：
    1. 系统检测触摸点是否落在预定义的“可触摸区域”碰撞体内（区域定义见 `Touch_Feedback_Map.md`）。
    2. 若命中，系统根据触摸区域ID，从配置表中查询对应的反馈组合（语音ID、表情ID、动作ID、Jiggle Physics强度等级）。
    3. 系统并行播放：语音（ASMR级别耳语）、表情动画（脸红、眨眼等）、动作（如角色微微后缩或侧头）、Jiggle Physics（胸部/臀部根据强度等级产生物理抖动）。
    4. 若触摸区域为“敏感区域”（如胸部、臀部），额外触发“害羞”状态：角色面部泛红程度+20%，持续3秒后逐渐消退。
- **【数据状态变化】**：
    - `PlayerDormitoryData.roomData[roleId].dailyInteractionCount += 1`（仅当触摸有效时增加）。
- **【表现层反馈】**：
    - 语音：通过耳机播放立体声耳语，音量比普通对话高5dB。
    - 表情：角色脸颊泛红（RGB值从(255,220,220)变为(255,180,180)），眼神变得迷离（瞳孔放大10%）。
    - 动作：被触摸部位对应的身体部分产生0.3秒的轻微后缩动画。
    - Jiggle Physics：胸部抖动幅度为5-15度（根据强度等级），臀部抖动幅度为3-8度。
- **【边界与异常兜底】**：
    - 若玩家连续快速点击同一区域（每秒超过3次），系统判定为“骚扰”，触发角色“生气”反馈：角色皱眉，语音变为“请温柔一点...”，并暂停触摸反馈1秒。
    - 若玩家触摸到非可触摸区域（如背景墙壁、家具），无任何反馈。
    - 若玩家在触摸反馈播放过程中切换视角，反馈动画继续播放，不受视角切换影响。

## 3.3 特殊姿势解锁

- **【前置条件】**：`PlayerDormitoryData.roomData[roleId].affectionLevel >= 5`（“坐在床边”姿势）；`>= 10`（“慵懒躺卧”姿势）。
- **【流转逻辑】**：
    1. 玩家在房间内点击“姿势切换”按钮（位于屏幕右下角，半透明）。
    2. 系统弹出姿势选择轮盘，显示已解锁的姿势图标。
    3. 玩家选择目标姿势，角色模型播放过渡动画（0.5秒），切换至新姿势。
    4. 切换后，摄像机自动调整至该姿势的推荐视角（如“慵懒躺卧”时，摄像机自动降低至床面高度）。
- **【数据状态变化】**：`PlayerDormitoryData.roomData[roleId].currentPoseId = 新姿势ID`。
- **【表现层反馈】**：
    - 过渡动画：角色从站立姿态平滑过渡至坐姿或躺姿，衣物物理模拟同步更新。
    - 推荐视角：摄像机以0.3秒的缓动曲线移动至新位置。
- **【边界与异常兜底】**：
    - 若玩家在姿势切换过程中断线，下次登录时，角色姿势恢复为默认站立姿势。
    - 若玩家在“慵懒躺卧”姿势下进行触摸互动，触摸区域映射表自动切换为躺卧版本（如胸部区域位置偏移）。

## 3.4 服装反馈联动

- **【前置条件】**：角色当前装备的皮肤ID在 `Skin_SpecialTouch_Config.json` 中有对应配置。
- **【流转逻辑】**：
    1. 玩家触摸角色时，系统除查询通用触摸反馈外，额外查询 `Skin_SpecialTouch_Config.json` 中该皮肤ID对应的触摸反馈覆盖配置。
    2. 若存在覆盖配置，则优先使用皮肤专属的语音、表情、动作和Jiggle Physics参数。
    3. 若不存在覆盖配置，则使用通用触摸反馈。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 泳装皮肤示例：触摸胸部时，语音变为“啊...好害羞...”，表情泛红程度+30%，Jiggle Physics强度提升至等级3（通用为等级2）。
    - 女仆皮肤示例：触摸手部时，角色做出“提裙角”动作，语音变为“主人，有什么吩咐吗？”。
- **【边界与异常兜底】**：
    - 若皮肤专属配置中引用的语音/动画资源缺失，系统自动降级为通用反馈，并在后台输出错误日志。
    - 皮肤专属反馈不得违反“擦边底线”规则（见3.5节）。

## 3.5 底线遵守

- **【硬性规则】**：
    1. 所有触摸反馈**绝不涉及**脱衣、露点或模拟性行为的动画或逻辑。
    2. 所有剧情对话**止步于**“恋人未满”或“极度暧昧”的尺度。禁止出现直接表白、接吻、同床等明确恋爱关系确认的台词。
    3. Jiggle Physics 的抖动幅度上限为15度，且仅限胸部、臀部区域。禁止对私密部位（如大腿内侧、裆部）设置可触摸区域。
    4. 所有“敏感区域”的触摸反馈，必须包含“害羞”状态（泛红、眼神迷离），以强调“这是暧昧互动，而非性行为”的定位。
- **【审查机制】**：所有语音、动画、对话文本在入库前，必须经过 `Audit Agent` 的合规审查，审查标准参照《内容红线文档》。

# 四、经济循环与商业化埋点

## 4.1 付费点

- **【核心买断】**：
    - **商品ID**：`shop_item_dormitory_unlock`
    - **定价**：¥30 / $4.99
    - **购买逻辑**：一次性付费，永久解锁 `PlayerDormitoryData.isUnlocked`。
- **【皮肤驱动】**：
    - 宿舍系统是展示新皮肤的核心场景。高品质皮肤（含专属宿舍互动姿势、语音）定价为 ¥68 / $9.99（比普通皮肤高30%）。
    - 皮肤详情页增加“宿舍预览”按钮，允许玩家在购买前，在宿舍场景中预览皮肤效果（限时30秒）。
- **【家具礼包】**：
    - **商品ID**：`shop_item_furniture_pack_01`（和风）、`shop_item_furniture_pack_02`（现代简约）
    - **定价**：¥12 / $1.99 每套
    - **购买逻辑**：购买后，对应家具套装ID被添加到 `PlayerDormitoryData.roomData[roleId].unlockedFurnitureSets` 列表中。玩家可在房间装修界面自由切换已解锁的家具套装。

## 4.2 资源产出

- **【好感度经验】**：
    - 每日有效触摸次数上限为50次，每次产出10点好感度经验。
    - 每日有效对话次数上限为5次，每次产出50点好感度经验。
    - 每日好感度经验获取上限：50 * 10 + 5 * 50 = 750点。
- **【无抽卡资源产出】**：该系统不产出任何抽卡资源（如钻石、招募券），以维持其“纯情感陪伴”的定位。

# 五、旧系统与数据联动 (强依赖)

## 5.1 角色系统

- **【调用接口】**：`GetRoleModel(roleId)`、`GetRoleCurrentSkinId(roleId)`
- **【联动逻辑】**：进入房间时，系统调用上述接口获取角色模型和当前皮肤，加载到房间场景中。

## 5.2 好感度系统

- **【调用接口】**：`GetAffectionLevel(roleId)`、`AddAffectionExp(roleId, exp)`
- **【联动逻辑】**：
    - 宿舍内触摸和对话产出的好感度经验，通过 `AddAffectionExp` 接口写入全局好感度系统。
    - 宿舍内解锁的对话和姿势，依赖 `GetAffectionLevel` 返回的等级进行判断。

## 5.3 商城系统

- **【调用接口】**：`PurchaseItem(playerId, itemId)`
- **【联动逻辑】**：玩家购买“私人宿舍”模块或家具礼包时，调用此接口。购买成功后，系统根据 `itemId` 更新 `PlayerDormitoryData`。

## 5.4 新增全局接口

- **【数据结构定义】**：
    ```json
    {
      "PlayerDormitoryData": {
        "isUnlocked": false, // bool, 是否已购买解锁
        "roomData": {
          // key: roleId (string), value: 角色房间数据
          "role_001": {
            "affectionLevel": 1, // int, 当前好感度等级
            "affectionExp": 0, // int, 当前好感度经验值
            "dailyInteractionCount": 0, // int, 今日有效触摸次数
            "dailyDialogueCount": 0, // int, 今日有效对话次数
            "currentPoseId": "pose_stand", // string, 当前姿势ID
            "currentFurnitureSetId": "furniture_default", // string, 当前家具套装ID
            "unlockedFurnitureSets": ["furniture_default"], // array<string>, 已解锁家具套装列表
            "unlockedDialogues": [], // array<string>, 已解锁对话ID列表
            "unlockedPoses": ["pose_stand"] // array<string>, 已解锁姿势ID列表
          }
        }
      }
    }
    ```
- **【接口定义】**：
    - `GetPlayerDormitoryData(playerId)`：返回 `PlayerDormitoryData` 对象。
    - `UpdateRoomCustomization(playerId, roleId, furnitureSetId)`：更新指定角色的家具套装。
    - `RecordDailyInteraction(playerId, roleId)`：记录一次有效触摸，增加 `dailyInteractionCount`。
    - `RecordDailyDialogue(playerId, roleId)`：记录一次有效对话，增加 `dailyDialogueCount`。

## 7.1 进入房间运镜UI

## 7.2 极简互动UI

- **按钮样式**：所有按钮（返回、对话、姿势切换、设置）默认状态为半透明圆形图标（直径：屏幕宽度的8%，背景色：#000000，透明度：30%，图标颜色：#FFFFFF，透明度：70%）。
- **隐藏规则**：玩家无操作3秒后，所有按钮以0.5秒淡出动画隐藏。玩家触摸屏幕任意位置（非按钮区域）时，按钮以0.3秒淡入动画恢复显示。
- **好感度进度条**：位于屏幕顶部中央，宽度为屏幕宽度的60%，高度为屏幕高度的1.5%。背景色：#333333，填充色：#FFD700。进度条两端显示当前经验值/升级所需经验值（字体：思源黑体 Regular，字号：屏幕宽度的2.5%，颜色：#FFFFFF）。

- **触发时**：在触摸点附近显示一个微小的粉色心形图标（直径：屏幕宽度的3%），以0.5秒淡入，停留1秒后以0.5秒淡出。
- **敏感区域触摸时**：心形图标变为深红色，且尺寸增大至屏幕宽度的5%。
- **“骚扰”判定时**：在屏幕中央显示一个红色感叹号图标（直径：屏幕宽度的10%），以0.3秒闪烁3次后消失，同时播放“叮”的警告音效。

# 八、性能与兼容性要求

## 8.1 最低配置要求

- **iOS**：iPhone 8 及以上，iOS 13.0 及以上，RAM 2GB。
- **Android**：骁龙 835 / 麒麟 970 / 天玑 700 及以上，RAM 3GB，Android 9.0 及以上。
- **渲染要求**：支持 OpenGL ES 3.0 或 Vulkan 1.0。

## 8.2 性能降级策略

- **动态光影**：在最低配置设备上，自动切换为预烘焙静态光照贴图，关闭实时阴影。
- **Jiggle Physics**：在最低配置设备上，Jiggle Physics 的物理模拟帧率从60fps降为30fps，抖动幅度减半。
- **角色模型LOD**：在摄像机距离角色超过3米时，切换为LOD1模型（面数减少50%）；超过5米时，切换为LOD2模型（面数减少70%）。

## 8.3 加载优化

- **场景预加载**：在玩家进入角色选择界面时，后台预加载当前选中角色的房间场景资源（模型、贴图、音频）。
- **资源缓存**：已加载过的房间场景资源，在玩家退出后保留在内存中30分钟，以便快速重新进入。

# 十一、风险与缓解措施

## 11.1 触摸反馈实现阻塞点

- **风险**：`Code Agent` 在实现触摸反馈时，需要与动画/音频团队协作获取具体资源。若资源未制作完成，会导致代码无法联调。
- **缓解**：在 `Touch_Feedback_Map.md` 中使用占位符ID，并提前与资源制作团队对齐资源交付时间表。

## 11.2 UI极简化与触摸反馈冲突

- **风险**：极简UI可能遮挡触摸区域，或触摸反馈的UI提示与极简原则冲突。
- **缓解**：明确UI元素的层级（z-order）和碰撞体穿透规则。触摸反馈提示设计为可配置的（如：仅显示图标，不显示文字）。

## 11.3 好感度系统与旧系统数据联动冲突

- **风险**：旧好感度系统可能未预留“宿舍专属对话”或“宿舍专属姿势”的接口。
- **缓解**：在系统设计文档中明确标注需要新增的全局接口，并提前与旧系统负责人沟通接口变更计划。

## 11.4 “擦边底线”审查标准不一致

- **风险**：`Audit Agent` 对“擦边底线”的理解可能与 `System Planner` 和 `Combat Agent` 不一致，导致反复驳回。
- **缓解**：在项目启动前，与所有相关Agent共同制定一份明确的“内容红线文档”，作为审查的硬性标准。

# [1.4 核心互动：触摸反馈系统] 修正版

- **【前置条件】**：玩家处于可控视角模式，且手指点击到角色身体碰撞体；`PlayerDormitoryData.roomData[roleId].dailyInteractionCount < 50`。
- **【流转逻辑】**：
    1. 系统检测触摸点是否落在预定义的“可触摸区域”碰撞体内（区域定义见 `Touch_Feedback_Map.md`）。
    2. 若命中，系统检查 `PlayerDormitoryData.roomData[roleId].dailyInteractionCount` 字段值是否小于50。
    3. 若小于50，判定为有效触摸，执行以下步骤：
        a. 系统根据触摸区域ID，从配置表中查询对应的反馈组合（语音ID、表情ID、动作ID、Jiggle Physics强度等级）。
        b. 系统并行播放：语音（ASMR级别耳语）、表情动画（脸红、眨眼等）、动作（如角色微微后缩或侧头）、Jiggle Physics（胸部/臀部根据强度等级产生物理抖动）。
        c. 若触摸区域为“敏感区域”（如胸部、臀部），额外触发“害羞”状态：角色面部泛红程度+20%，持续3秒后逐渐消退。
        d. 更新数据：`PlayerDormitoryData.roomData[roleId].dailyInteractionCount += 1`。
    4. 若 `PlayerDormitoryData.roomData[roleId].dailyInteractionCount` 已等于50，判定为无效触摸，不播放任何反馈，仅在UI上显示“今日互动次数已用尽”的灰色文字提示（持续2秒后消失）。
- **【数据状态变化】**：
    - 有效触摸：`PlayerDormitoryData.roomData[roleId].dailyInteractionCount += 1`；`PlayerDormitoryData.roomData[roleId].affectionExp += 10`。
    - 无效触摸（次数已满）：无数据变化。
- **【表现层反馈】**：
    - 有效触摸：播放对应语音、表情动画、Jiggle Physics；触摸点附近显示粉色心形图标。
    - 无效触摸（次数已满）：在屏幕中央显示“今日互动次数已用尽”灰色文字（字体：思源黑体 Regular，字号：屏幕宽度的4%，颜色：#888888），持续2秒后淡出。
- **【边界与异常兜底】**：
    - 若玩家在触摸反馈播放过程中断线，当前触摸的 `dailyInteractionCount` 增加操作以服务器最终保存的数据为准。若服务器未收到本次触摸请求，则次数不增加。
    - 若玩家在触摸反馈播放过程中切换视角，反馈动画继续播放，不受视角切换影响。
    - 若玩家连续快速点击同一区域（每秒超过3次），系统判定为“骚扰”，触发角色“生气”反馈，且该次触摸不计入 `dailyInteractionCount`。
    - 若玩家触摸到非可触摸区域（如背景墙壁、家具），无任何反馈，且不计入 `dailyInteractionCount`。
