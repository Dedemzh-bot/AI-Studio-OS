# 温泉中心 - 系统详细设计案

## 一、系统概述与设计愿景

### 1.1 系统定位
- **【前置条件】**：玩家必须已解锁宿舍系统，且至少拥有1名角色。
- **【流转逻辑】**：玩家从宿舍场景内点击“温泉”入口按钮，加载温泉中心场景。
- **【数据状态变化】**：加载完成后，系统记录玩家当前进入温泉中心的时间戳至 `last_hotspring_enter_timestamp`。
- **【边界与异常兜底】**：若玩家未解锁宿舍系统，温泉入口按钮不显示。若玩家拥有0名角色，入口按钮置灰并提示“请先获取一名角色”。

### 1.2 设计目标
- **【前置条件】**：系统需与宿舍系统、角色获取系统、图鉴系统、好感度系统完成数据对接。
- **【流转逻辑】**：温泉中心作为宿舍系统的扩展场景，不产出任何战斗数值、抽卡资源或养成材料。所有产出仅限于角色特殊语音/表情/动作的解锁、温泉中心专属的装饰/道具（纯观赏）。
- **【数据状态变化】**：所有温泉中心数据（角色刷新记录、互动记录、小游戏进度）归属于玩家账号全局存档，与角色好感度系统弱关联（仅记录互动行为，不直接产出好感度经验）。
- **【边界与异常兜底】**：若玩家在温泉中心中尝试通过非正常手段获取战斗数值或抽卡资源，系统记录异常日志并拒绝操作。

## 二、核心规则与玩法机制

### 2.1 角色刷新规则
- **【前置条件】**：玩家已进入温泉中心场景。
- **【流转逻辑】**：
  1. 系统从玩家已拥有的角色池中随机抽取1名角色。
  2. 若玩家当日首次进入温泉中心，且拥有“今日特惠角色”（由系统每日刷新指定），则必定刷新该角色。
  3. 角色以“泡澡”状态出现在场景中，模型为泳装/浴巾装扮，半身浸入水中。
  4. 玩家可主动点击“换人”按钮，消耗 `[REFRESH_CURRENCY_COST]` 基础货币重新随机刷新角色。
- **【数据状态变化】**：
  - 刷新成功后，`current_hotspring_character_id` 更新为新角色ID。
  - 若为“今日特惠角色”刷新，`daily_special_character_used` 置为 `true`。
  - 每次主动换人，`daily_free_refresh_count` 增加 `1`（若未超过 `[DAILY_FREE_REFRESH_LIMIT]`），否则扣除 `[REFRESH_CURRENCY_COST]`。
- **【表现层反馈】**：场景淡入，角色从水中站起/坐下，播放入水动画。换人时，当前角色消失，新角色从水中浮现。
- **【边界与异常兜底】**：
  - 若玩家已使用“今日特惠角色”机会，则当日不再触发。
  - 若 `daily_free_refresh_count` 已达 `[DAILY_FREE_REFRESH_LIMIT]` 且基础货币不足，换人按钮置灰并提示“货币不足”。
  - 若玩家角色池为空，场景显示空温泉，提示“暂无角色”。

### 2.2 互动小游戏 - 水枪射击靶子（轻度休闲解压类）
- **【前置条件】**：玩家已进入温泉中心，且当前有角色在场景中。
- **【流转逻辑】**：
  1. 玩家点击“水枪射击”小游戏入口。
  2. 场景中随机生成 `[TARGET_COUNT]` 个漂浮靶子（如：水球、浮标、鸭子），位置随机分布在水面上。
  3. 计时器开始倒计时 `[SHOOTING_TIME_LIMIT]` 秒。
  4. 玩家使用屏幕准星瞄准靶子，点击射击。命中判定为射线检测，命中后靶子爆裂/消失。
  5. 计时结束后，系统统计命中数。
  6. 若命中数 >= `[SHOOTING_SUCCESS_THRESHOLD]`，触发角色特殊奖励（解锁专属表情/语音）。
  7. **若玩家连续 `[FAILURE_THRESHOLD]` 次未达标（即命中数 < `[SHOOTING_SUCCESS_THRESHOLD]`），弹出“跳过”选项**。玩家可选择消耗 `[SKIP_CURRENCY_COST]` 基础货币直接获得奖励。
  8. 小游戏结束后，返回温泉中心主场景。
- **【数据状态变化】**：
  - 小游戏开始：`minigame_active` 置为 `true`，`minigame_type` 置为 `[MINIGAME_SHOOTING]`。
  - 每次射击命中：`shooting_hit_count` 增加 `1`。
  - 小游戏结束：`minigame_active` 置为 `false`，`minigame_type` 置为 `null`。
  - 若命中数达标或使用跳过：`character_minigame_reward_unlocked` 增加对应奖励ID，`daily_minigame_reward_count` 增加 `1`。
  - **`minigame_failure_count` 字段重置逻辑**：
    - 当玩家退出当前小游戏（无论成功或失败）时，`minigame_failure_count` 重置为 `0`。
    - 当玩家切换小游戏类型（从水枪射击切换到捞水球，或反之）时，`minigame_failure_count` **重置为 `0`**。
    - 当玩家切换角色时，`minigame_failure_count` 重置为 `0`。
- **【表现层反馈】**：镜头自动推近至角色面部特写，展示表情变化。靶子被击中时播放水花溅射特效，角色播放被水花溅到后的惊讶/害羞表情+语音。计时器与得分板以半透明毛玻璃UI显示。跳过选项弹窗显示消耗预览。
- **【边界与异常兜底】**：
  - 若 `daily_minigame_reward_count` 已达 `[DAILY_MINIGAME_REWARD_LIMIT]`，小游戏入口按钮置灰并提示“今日奖励已领完”。
  - 若玩家在游戏过程中断线，游戏状态保存至 `minigame_snapshot`，重连后可继续（若计时未结束）。
  - 若玩家主动退出小游戏，不消耗任何资源，`minigame_failure_count` 重置为 `0`。

### 2.3 互动小游戏 - 捞水球（轻度休闲解压类）
- **【前置条件】**：玩家已进入温泉中心，且当前有角色在场景中。
- **【流转逻辑】**：
  1. 玩家点击“捞水球”小游戏入口。
  2. 场景中出现一个水池，水面漂浮 `[BALL_COUNT]` 个彩色水球。
  3. 计时器开始倒计时 `[FISHING_TIME_LIMIT]` 秒。
  4. 玩家通过点击/拖拽操作，将水球捞至岸边指定区域。每次成功捞取，水球破裂。
  5. 计时结束后，系统统计捞取数。
  6. 若捞取数 >= `[FISHING_SUCCESS_THRESHOLD]`，触发角色特殊奖励（解锁专属表情/语音）。
  7. **若玩家连续 `[FAILURE_THRESHOLD]` 次未达标（即捞取数 < `[FISHING_SUCCESS_THRESHOLD]`），弹出“跳过”选项**。玩家可选择消耗 `[SKIP_CURRENCY_COST]` 基础货币直接获得奖励。
  8. 小游戏结束后，返回温泉中心主场景。
- **【数据状态变化】**：
  - 小游戏开始：`minigame_active` 置为 `true`，`minigame_type` 置为 `[MINIGAME_FISHING]`。
  - 每次成功捞取：`fishing_catch_count` 增加 `1`。
  - 小游戏结束：`minigame_active` 置为 `false`，`minigame_type` 置为 `null`。
  - 若捞取数达标或使用跳过：`character_minigame_reward_unlocked` 增加对应奖励ID，`daily_minigame_reward_count` 增加 `1`。
  - **`minigame_failure_count` 字段重置逻辑**：
    - 当玩家退出当前小游戏（无论成功或失败）时，`minigame_failure_count` 重置为 `0`。
    - 当玩家切换小游戏类型（从水枪射击切换到捞水球，或反之）时，`minigame_failure_count` **重置为 `0`**。
    - 当玩家切换角色时，`minigame_failure_count` 重置为 `0`。
- **【表现层反馈】**：镜头自动推近至角色面部特写，展示表情变化。水球被捞取时播放水花溅射特效，角色播放开心鼓掌/微笑表情+语音。计时器与进度条以半透明毛玻璃UI显示。跳过选项弹窗显示消耗预览。
- **【边界与异常兜底】**：
  - 若 `daily_minigame_reward_count` 已达 `[DAILY_MINIGAME_REWARD_LIMIT]`，小游戏入口按钮置灰并提示“今日奖励已领完”。
  - 若玩家在游戏过程中断线，游戏状态保存至 `minigame_snapshot`，重连后可继续（若计时未结束）。
  - 若玩家主动退出小游戏，不消耗任何资源，`minigame_failure_count` 重置为 `0`。

### 2.4 奖励系统
- **【前置条件】**：玩家在小游戏中达到奖励条件（命中数/捞取数达标，或使用跳过）。
- **【流转逻辑】**：
  1. 系统检查当前角色在温泉中心场景下的奖励池。
  2. 从奖励池中随机选取一个未解锁的奖励（专属表情/语音/动作）。
  3. 奖励永久解锁，可在图鉴系统中回看。
- **【数据状态变化】**：`character_hotspring_rewards_unlocked` 增加对应奖励ID。`daily_minigame_reward_count` 增加 `1`。
- **【表现层反馈】**：弹窗显示解锁内容预览（表情/语音/动作名称与缩略图），播放解锁特效。
- **【边界与异常兜底】**：
  - 若当前角色所有奖励已全部解锁，小游戏入口按钮置灰并提示“该角色奖励已全部解锁”。
  - 若奖励池为空（无可用奖励），系统提示“暂无可用奖励”。

## 三、表现层与角色展示联动（核心强制项）

### 3.1 场景与运镜
- **【前置条件】**：玩家进入温泉中心场景。
- **【流转逻辑】**：场景为日式露天温泉风格，背景为山景/竹林，水面有蒸汽粒子特效。角色半身浸入水中，水面以上可见上半身、手臂、头发。模型为泳装/浴巾装扮，头发有湿发效果。
- **【数据状态变化】**：`camera_mode` 切换为 `[CAMERA_MODE_HOTSPRING_DEFAULT]`。
- **【表现层反馈】**：默认镜头为第三人称近肩视角，略微俯视。玩家可自由旋转/缩放镜头，但限制最低角度（防止穿模或过度暴露）。小游戏触发时，镜头自动推近至角色面部特写。小游戏结束后镜头平滑复位。
- **【边界与异常兜底】**：若玩家手动调整视角超出限制范围，镜头自动回弹至合法角度。

### 3.2 物理反馈
- **【前置条件】**：角色模型搭载物理骨骼。
- **【流转逻辑】**：角色移动/互动时，水面产生波纹与涟漪。水球/靶子被击中时，水花溅射特效。头发、浴巾边缘有物理抖动。被水花溅到时，角色有轻微后仰/躲闪动画。
- **【数据状态变化】**：无后台数据变化。
- **【表现层反馈】**：物理效果实时渲染，角色表情根据互动类型切换（开心、惊讶、害羞、微笑）。
- **【边界与异常兜底】**：若设备性能不足，物理效果降级为预设动画，水面粒子特效关闭。

### 3.3 UI透明化
- **【前置条件】**：玩家进入温泉中心场景。
- **【流转逻辑】**：所有UI组件（小游戏计时器、得分、按钮）采用半透明毛玻璃设计，不遮挡角色主体。小游戏进行时，UI自动隐藏，仅保留准星/操作提示。玩家可一键隐藏所有UI，进入纯观赏模式。
- **【数据状态变化】**：`ui_visibility_state` 切换为 `[UI_VISIBLE]` 或 `[UI_HIDDEN]`。
- **【表现层反馈】**：UI透明度变化，隐藏后仅保留角色与场景。
- **【边界与异常兜底】**：若玩家在隐藏UI状态下触发小游戏，UI自动短暂显示后再次隐藏。

### 3.4 擦边底线
- **【前置条件】**：角色泳装/浴巾装扮严格符合游戏分级标准。
- **【流转逻辑】**：水面高度控制在胸部以下，不展示隐私部位。角色反馈（被水花溅到、被瞄准）均为“可爱/害羞”反应，禁止任何性暗示动作或台词。温泉中心不开放触摸互动功能。
- **【数据状态变化】**：无后台数据变化。
- **【表现层反馈】**：角色表情与动作严格控制在“可爱/害羞”范围内。
- **【边界与异常兜底】**：若玩家尝试通过非正常手段触发触摸互动，系统记录异常日志并拒绝操作。

## 四、经济循环与商业化埋点

### 4.1 免费产出
- **【前置条件】**：玩家每日首次进入温泉中心。
- **【流转逻辑】**：系统检查 `daily_special_character_used`，若为 `false` 则提供1次“今日特惠角色”刷新机会。
- **【数据状态变化】**：`daily_special_character_used` 置为 `true`。
- **【表现层反馈】**：场景中显示“今日特惠角色”标识。
- **【边界与异常兜底】**：若玩家当日已使用过特惠机会，不再触发。

### 4.2 付费埋点
- **【前置条件】**：玩家在商城中购买温泉中心专属皮肤、刷新券或小游戏次数重置券。
- **【流转逻辑】**：购买后，道具进入玩家背包。玩家可在温泉中心中使用。
- **【数据状态变化】**：背包中道具数量增加，玩家付费货币减少。
- **【表现层反馈】**：购买成功弹窗提示，道具图标出现在背包中。
- **【边界与异常兜底】**：若玩家付费货币不足，购买按钮置灰并提示“货币不足”。

### 4.3 资源循环
- **【前置条件】**：温泉中心不产出任何抽卡资源、战斗属性、养成材料。
- **【流转逻辑】**：所有产出均为纯观赏/陪伴向内容（表情、语音、动作解锁）。基础货币消耗：刷新角色、购买小游戏次数重置券。
- **【数据状态变化】**：无抽卡资源相关数据变化。
- **【表现层反馈】**：无抽卡资源相关UI显示。
- **【边界与异常兜底】**：若玩家尝试通过温泉中心系统获取抽卡资源，系统提示“该功能不产出抽卡资源”。

## 五、旧系统与数据联动

### 5.1 宿舍系统
- **【前置条件】**：温泉中心作为宿舍系统的扩展场景。
- **【流转逻辑】**：入口位于宿舍场景内。宿舍系统的“换装”功能可切换温泉中心中角色的泳装皮肤（需已购买）。宿舍系统的“拍照模式”在温泉中心中同样可用（但限制镜头角度）。
- **【数据状态变化】**：`hotspring_center_unlocked` 置为 `true`。
- **【表现层反馈】**：宿舍场景中出现“温泉”入口按钮。
- **【边界与异常兜底】**：若玩家未解锁宿舍系统，温泉入口不显示。

### 5.2 角色获取系统
- **【前置条件】**：温泉中心角色池 = 玩家已拥有的所有角色。
- **【流转逻辑】**：新角色获取后，自动加入温泉中心角色池（无需额外解锁）。
- **【数据状态变化】**：`character_hotspring_available` 置为 `true`。
- **【表现层反馈】**：温泉中心角色刷新池更新。
- **【边界与异常兜底】**：若玩家重复获得已拥有角色，不重复加入角色池。

### 5.3 图鉴系统
- **【前置条件】**：玩家在温泉中心中解锁特殊表情/语音/动作。
- **【流转逻辑】**：解锁内容自动同步至图鉴系统，玩家可在图鉴中回看。图鉴中新增“温泉中心”分类，展示所有已解锁内容。
- **【数据状态变化】**：`collection_book_hotspring_entry_unlocked` 置为 `true`。
- **【表现层反馈】**：图鉴界面显示已解锁内容，未解锁内容显示为锁图标。
- **【边界与异常兜底】**：若玩家未解锁内容，图鉴中不显示。

### 5.4 好感度系统
- **【前置条件】**：温泉中心互动行为不直接产出好感度经验。
- **【流转逻辑】**：温泉中心解锁的特殊内容（表情/语音）可在宿舍系统中使用（如：触摸互动时触发温泉中心专属语音）。
- **【数据状态变化】**：`character_hotspring_interaction_record` 记录互动次数（仅用于统计，不产出经验）。
- **【表现层反馈】**：宿舍触摸互动时，若角色有温泉中心专属语音，优先播放。
- **【边界与异常兜底】**：若角色无温泉中心专属语音，使用默认语音。

## 六、待确认风险与追问

### 6.1 性能风险
- **【前置条件】**：温泉中心场景需同时加载角色模型（泳装/浴巾）、水面物理、蒸汽粒子、小游戏逻辑。
- **【流转逻辑】**：初期仅允许1个角色同时出现，后续根据性能优化情况开放多角色同池。
- **【数据状态变化】**：`max_simultaneous_characters` 初始为 `1`。
- **【表现层反馈】**：场景中仅显示当前选中角色。
- **【边界与异常兜底】**：若设备性能不足，自动降低水面物理与粒子特效精度。

### 6.2 内容生产量
- **【前置条件】**：每个角色需要独立制作：泳装/浴巾模型、湿发效果、5-8条温泉专属语音、3-5个表情、2个小游戏反馈动画。
- **【流转逻辑】**：首批仅覆盖3-5个高人气角色，后续角色分批更新。
- **【数据状态变化】**：`content_production_phase` 记录当前生产阶段。
- **【表现层反馈】**：未覆盖角色显示“即将上线”提示。
- **【边界与异常兜底】**：若美术/音频团队产能不足，系统可临时使用通用动作与语音。

### 6.3 小游戏平衡性
- **【前置条件】**：水枪射击与捞水球的难度需适中。
- **【流转逻辑】**：确保大部分玩家可在1-2次尝试内完成。若玩家多次失败，提供“跳过”选项（消耗基础货币）。
- **【数据状态变化】**：`minigame_difficulty_adjustment` 记录难度调整日志。
- **【表现层反馈】**：跳过选项弹窗显示消耗预览。
- **【边界与异常兜底】**：若玩家跳过次数过多，系统提示“跳过次数已达上限”。
