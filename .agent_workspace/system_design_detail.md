# 好友系统 - 系统详细设计案

## 一、系统概述与设计愿景

- **【核心定位】**：轻度社交助战系统。好友系统是玩家获取额外战斗资源（借角色）的轻量级渠道，同时附带基础的情感维系交互，强化“分析员”与“战友”之间的陪伴感，而非构建复杂社交网络。
- **【设计愿景】**：让好友系统成为玩家日常资源循环的“润滑剂”，通过借角色降低养成门槛、提升通关体验；同时通过点赞、赠送体力等轻交互，营造“有人陪伴”的温暖氛围，契合项目ML向的情感基调。

## 二、核心规则与玩法机制

### 2.1 好友容量

- **【前置条件】**：玩家等级达到5级解锁好友系统。
- **【流转逻辑】**：
    1. 玩家进入好友界面，系统显示当前好友数量/上限。
    2. 初始上限为20人。
    3. 玩家可通过消耗“信用点”或达到特定等级解锁扩容至50人。
- **【数据状态变化】**：
    - 扩容消耗：第21-30位，每扩容1位消耗5000信用点；第31-40位，每扩容1位消耗10000信用点；第41-50位，每扩容1位消耗20000信用点。
    - 等级解锁：玩家等级达到15级时自动扩容至25人；达到25级时自动扩容至30人。
- **【表现层反馈】**：扩容成功时，UI弹出提示“好友上限已提升至X人”，并播放轻量级特效。
- **【边界与异常兜底】**：
    - 信用点不足时，扩容按钮置灰并显示“信用点不足”。
    - 已达50人上限时，无法继续扩容，按钮显示“已达上限”。

### 2.2 好友管理

- **【前置条件】**：玩家等级达到5级。
- **【流转逻辑】**：
    1. **申请好友**：玩家可通过搜索ID或从推荐列表点击“申请”按钮。
    2. **同意/拒绝**：对方在好友申请列表中可点击“同意”或“拒绝”。
    3. **删除好友**：在好友列表中长按或点击“删除”按钮，单方面删除，无需对方确认。
    4. **拉黑**：在好友列表中点击“拉黑”按钮，拉黑后双方不再是好友，且对方无法再向你发送好友申请。
- **【数据状态变化】**：
    - 申请成功：向对方发送一条待处理的好友申请记录。
    - 同意申请：双方好友关系建立，各自好友列表增加对方。
    - 拒绝申请：申请记录删除，不通知申请方。
    - 删除好友：双方好友关系解除，各自好友列表移除对方。
    - 拉黑：双方好友关系解除，申请方被加入黑名单，无法再发送申请。
- **【表现层反馈】**：
    - 申请成功：UI提示“好友申请已发送”。
    - 同意申请：申请方收到邮件通知“XX已同意你的好友申请”。
    - 删除好友：UI提示“已删除好友”，无通知。
    - 拉黑：UI提示“已拉黑该玩家”。
- **【边界与异常兜底】**：
    - 对方好友已满时，申请失败，提示“对方好友已满”。
    - 自己好友已满时，无法发送申请，提示“好友已满，请先扩容”。
    - 搜索ID不存在时，提示“未找到该玩家”。
    - 已拉黑的玩家再次搜索时，不显示该玩家。

### 2.3 推荐列表

- **【前置条件】**：玩家进入好友推荐界面。
- **【流转逻辑】**：
    1. 系统根据算法生成推荐列表，每次刷新显示10名玩家。
    2. 玩家可点击“刷新”按钮手动刷新列表，每日免费刷新3次，之后消耗20数据金/次。
- **【数据状态变化】**：
    - 推荐算法权重：等级相近（±3级）权重40%，在线状态（在线玩家）权重30%，助战角色强度（总战力）权重20%，随机权重10%。
    - 每日刷新次数记录在玩家数据中，每日0点重置。
- **【表现层反馈】**：推荐列表展示玩家头像、等级、看板娘、在线状态、助战角色战力。
- **【边界与异常兜底】**：
    - 免费刷新次数用尽后，刷新按钮变为消耗数据金模式。
    - 数据金不足时，刷新按钮置灰。

### 2.4 助战系统

- **【前置条件】**：玩家进入关卡选择界面，且该关卡允许使用助战。
- **【流转逻辑】**：
    1. 玩家在关卡选择界面点击“助战”入口。
    2. 系统展示好友列表，每个好友卡片显示其设置的“助战角色”信息（角色名、等级、战力）。
    3. 玩家点击选择一名好友的助战角色，确认后进入战斗。
    4. 助战角色作为第四人加入编队，不替换己方三人。
    5. 战斗结束后，根据战斗结果发放奖励。
- **【数据状态变化】**：
    - **借用方**：每日可借用助战5次，每次通关后获得5点友情点。
    - **借出方**：好友借用你的角色成功后，你获得10点友情点邮件通知，每日上限10次。
    - **助战角色死亡判定**：若助战角色在战斗中死亡，借用方友情点奖励减半（2点），但借出方仍获得全额10点友情点。助战角色死亡不影响关卡评价。
    - 每日借用次数和借出次数记录在玩家数据中，每日0点重置。
- **【表现层反馈】**：
    - 助战选择界面：展示好友助战角色的3D模型（可旋转查看），并播放“助战登场”动画（角色挥手、摆pose、语音：“分析员，我来帮你啦~”）。
    - 战斗结算界面：助战角色出现在队伍后方，与己方三名角色一同庆祝胜利，播放专属胜利动作和语音。
    - 借出方收到邮件通知：“你的XX角色被好友借用，获得10点友情点”。
- **【边界与异常兜底】**：
    - 每日借用次数用尽时，助战入口按钮置灰，提示“今日借用次数已用完”。
    - 每日借出次数用尽时，好友借用你的角色后，你不再获得友情点，但好友仍可借用。
    - 好友的助战角色正在被其他玩家借用时，该好友的助战角色仍可被选择（允许多人同时借用同一角色）。
    - 好友离线时，其设置的助战角色仍可被借用。
    - 高难挑战（如拟境、战术考核）中，助战入口不显示。

### 2.5 轻交互：赠送体力与点赞

- **【前置条件】**：玩家与好友已建立好友关系。
- **【流转逻辑】**：
    1. **赠送体力**：在好友列表中，点击好友卡片上的“赠送体力”按钮，每日限1次/人。
    2. **点赞**：在好友列表中，点击好友卡片上的“点赞”按钮，每日限1次/人。
- **【数据状态变化】**：
    - 赠送体力：消耗自身5点体力，好友获得5点体力邮件通知。
    - 点赞：无消耗，好友获得2点友情点邮件通知。
- **【表现层反馈】**：
    - 赠送体力/点赞时，触发轻量级UI弹窗，展示好友看板娘对你做出感谢动作（飞吻、比心、鞠躬），并播放感谢语音（如：“谢谢你，分析员~”）。
    - 按钮变为已赠送/已点赞状态，次日0点重置。
- **【边界与异常兜底】**：
    - 自身体力不足5点时，赠送体力按钮置灰，提示“体力不足”。
    - 已赠送/已点赞后，按钮置灰，不可重复操作。

### 2.6 友情点与商店

- **【前置条件】**：玩家拥有友情点。
- **【流转逻辑】**：
    1. 玩家进入友情商店界面，消耗友情点兑换商品。
    2. 商品列表包括：单抽券碎片、角色养成材料、信用点等。
- **【数据状态变化】**：
    - 友情点获取：每日登录赠送好友体力（5点/次，上限50点）、点赞（2点/次，上限20点）、助战被借用（10点/次，上限100点），每日总计约170点。
    - 友情商店定价：
        - 单抽券碎片（10个碎片合成1张单抽券）：100友情点/个，每月限购30个。
        - 角色经验材料（大）：50友情点/个，每日限购10个。
        - 角色突破材料（随机）：30友情点/个，每日限购5个。
        - 信用点：10友情点/1000信用点，每日限购20次。
- **【表现层反馈】**：购买成功时，UI弹出“购买成功”提示，并播放获得物品的特效。
- **【边界与异常兜底】**：
    - 友情点不足时，购买按钮置灰，提示“友情点不足”。
    - 限购次数用尽时，商品显示“已售罄”。

### 2.7 好友主页

- **【前置条件】**：玩家点击好友列表中的好友卡片。
- **【流转逻辑】**：
    1. 进入好友主页，展示好友的看板娘（3D模型，带动态呼吸/待机动作）。
    2. 展示好友签名、近期战绩（如通关最高难度关卡）、好感度最高的角色展示。
- **【数据状态变化】**：无。
- **【表现层反馈】**：好友主页全屏展示，看板娘可旋转查看。
- **【边界与异常兜底】**：好友删除你后，无法进入其主页。

## 三、表现层与角色展示联动 (核心强制项)

### 3.1 好友列表UI设计

- **【前置条件】**：玩家进入好友列表界面。
- **【流转逻辑】**：系统加载好友数据，生成列表卡片。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 每个好友卡片展示其“看板娘”的3D模型缩略图（半身或全身，带动态呼吸/待机动作）。
    - 卡片下方显示好友等级、在线状态、助战角色战力。
    - 卡片右侧显示“赠送体力”、“点赞”、“删除”等操作按钮。
- **【边界与异常兜底】**：好友离线时，看板娘模型仍显示，但在线状态图标变为灰色。

### 3.2 助战选择界面

- **【前置条件】**：玩家在关卡选择界面点击“助战”入口。
- **【流转逻辑】**：系统加载好友助战数据，生成选择列表。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 选择好友时，展示该好友助战角色的高清3D模型（可旋转查看）。
    - 播放专属的“助战登场”动画（角色挥手、摆pose、说一句语音：“分析员，我来帮你啦~”）。
    - 模型下方显示角色等级、技能等级、武器、后勤小队信息。
- **【边界与异常兜底】**：好友未设置助战角色时，该好友在助战列表中不显示。

### 3.3 特殊机位与互动

- **【前置条件】**：玩家执行赠送体力或点赞操作。
- **【流转逻辑】**：触发轻量级UI弹窗。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 弹窗展示好友看板娘对你做出感谢动作（飞吻、比心、鞠躬），并播放一句感谢语音（如：“谢谢你，分析员~”）。
    - 弹窗持续2秒后自动关闭。
- **【边界与异常兜底】**：好友离线时，感谢动作和语音仍可正常播放。

### 3.4 助战结算

- **【前置条件】**：战斗结束，进入结算界面。
- **【流转逻辑】**：系统加载助战角色数据，在结算界面展示。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 助战角色出现在队伍后方，与你的三名角色一同庆祝胜利。
    - 播放专属的胜利动作和语音（如：“合作愉快，分析员！”）。
- **【边界与异常兜底】**：助战角色死亡时，结算界面中该角色仍会出现，但动作变为虚弱状态（如单手撑地），语音变为“抱歉，没能帮上忙...”。

### 3.5 擦边暗示底线

- **【前置条件】**：所有互动动作和语音设计。
- **【流转逻辑】**：审核流程。
- **【数据状态变化】**：无。
- **【表现层反馈】**：
    - 所有互动动作（飞吻、比心、挥手）控制在“亲密战友”尺度，不涉及性暗示。
    - 语音台词严格审核，禁止暧昧或越界内容，保持“战友间的信赖与鼓励”基调。
- **【边界与异常兜底】**：任何违反底线的设计将被驳回修改。

## 四、经济循环与商业化埋点

### 4.1 经济循环

- **【前置条件】**：玩家每日进行好友交互。
- **【流转逻辑】**：每日通过好友交互产出友情点，消耗于友情商店。
- **【数据状态变化】**：
    - 每日友情点产出上限：约170点（赠送体力50点 + 点赞20点 + 被借用100点）。
    - 每月友情点产出上限：约5100点。
    - 每月友情点消耗：单抽券碎片（3000点） + 养成材料（约1500点） = 约4500点。
- **【表现层反馈】**：友情点数量在商店界面实时更新。
- **【边界与异常兜底】**：友情点溢出时，超出上限的部分不记录，但玩家仍可继续获取（不浪费）。

### 4.2 商业化埋点

- **【前置条件】**：玩家需要扩容好友位或购买皮肤。
- **【流转逻辑】**：
    1. **好友位扩容**：初始20人，后续扩容至50人需消耗信用点或数据金。
    2. **助战角色皮肤展示**：好友借用你的角色时，会展示你当前为该角色穿戴的皮肤。
    3. **好友邀请奖励**：邀请好友注册并达到一定等级，邀请者获得限定头像框、少量数据金等奖励。
- **【数据状态变化】**：
    - 扩容消耗：第21-30位，每扩容1位消耗5000信用点或50数据金；第31-40位，每扩容1位消耗10000信用点或100数据金；第41-50位，每扩容1位消耗20000信用点或200数据金。
    - 邀请奖励：被邀请者达到10级，邀请者获得限定头像框；达到20级，获得100数据金；达到30级，获得200数据金。
- **【表现层反馈】**：皮肤展示在助战选择界面和好友列表中。
- **【边界与异常兜底】**：被邀请者已注册时，无法再次邀请。

## 五、旧系统与数据联动 (强依赖)

### 5.1 角色系统

- **【前置条件】**：好友设置助战角色。
- **【流转逻辑】**：好友助战角色数据直接读取好友的角色养成状态。
- **【数据状态变化】**：好友的角色等级、技能等级、武器、后勤小队数据实时同步。
- **【表现层反馈】**：好友列表展示的看板娘数据来自好友的“大厅展示”配置。
- **【边界与异常兜底】**：好友更换助战角色后，你的助战列表在下次进入时更新。

### 5.2 战斗系统

- **【前置条件】**：玩家选择助战角色进入战斗。
- **【流转逻辑】**：关卡选择界面新增“助战”入口，调用好友列表数据。
- **【数据状态变化】**：战斗结算逻辑需处理助战角色的伤害统计、死亡判定。
- **【表现层反馈】**：助战角色在战斗中正常行动。
- **【边界与异常兜底】**：助战角色死亡不影响玩家评价，但会减少助战奖励。

### 5.3 邮件系统

- **【前置条件】**：好友借用角色、赠送体力等行为。
- **【流转逻辑】**：触发邮件通知。
- **【数据状态变化】**：邮件系统需支持“一键领取”所有好友相关奖励。
- **【表现层反馈】**：邮件通知包含奖励内容和来源。
- **【边界与异常兜底】**：邮件过期后，奖励无法领取。

### 5.4 成就/任务系统

- **【前置条件】**：玩家完成好友相关操作。
- **【流转逻辑】**：新增日常任务和成就。
- **【数据状态变化】**：
    - 日常任务：赠送好友体力3次（奖励10点友情点）、借用助战1次（奖励5点友情点）。
    - 成就：累计借用助战100次（奖励100数据金）、累计被借用500次（奖励200数据金）。
- **【表现层反馈】**：任务进度在任务界面实时更新。
- **【边界与异常兜底】**：任务每日0点重置。

## 六、待确认风险与追问

- **【风险】**：助战系统可能降低玩家养成动力（过度依赖好友强力角色）。需通过限制每日借用次数、高难本禁用助战来平衡。
- **【追问】**：是否需要在好友系统中加入“聊天”功能（预设短语/表情包）？当前草案未包含，如需加入可后续扩展。
