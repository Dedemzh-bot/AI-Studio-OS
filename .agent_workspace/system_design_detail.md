### 系统概述与设计愿景
- **【前置条件】**：无。该模块是游戏核心体验的组成部分，与主线进度、玩家等级无关。
- **【流转逻辑】**：
    1. 玩家完成新手引导后，系统自动激活【角色私人宿舍】入口。
    2. 入口位于主界面“基地”功能模块内，以独立图标呈现。
    3. 玩家点击进入后，系统展示已获得角色列表（按稀有度、获取时间排序）。
    4. 玩家选择任一角色，系统加载其专属宿舍场景与高精度3D模型。
- **【数据状态变化】**：无。该模块不产生任何与养成、资源相关的数据变化。
- **【表现层反馈】**：主界面“基地”入口出现微光动效，提示新功能解锁。首次进入特定角色宿舍时，播放一段15秒的专属入场动画（角色从床上醒来或从窗边转身）。
- **【边界与异常兜底】**：
    - **无角色状态**：若玩家尚未获得任何角色，宿舍入口图标置灰，点击后弹窗提示“请先通过抽卡或任务获得一位战术人形，她正在等你回家。”，并提供跳转至抽卡界面的按钮。
    - **网络波动**：加载宿舍场景时若网络中断，界面显示加载进度条并提示“正在努力加载中...”，超时30秒后弹窗提示“网络连接不稳定，请检查后重试”，并提供“重试”与“返回主界面”按钮。

### 核心规则与玩法机制
- **【前置条件】**：玩家已获得该角色（不限稀有度、星级）。
- **【流转逻辑】**：
    1. **进入**：在角色列表中选择角色，点击“进入宿舍”按钮。
    2. **自由探索**：进入后，默认采用第一人称近景视角。玩家可通过单指拖拽旋转视角，双指捏合缩放视角。
    3. **交互触发**：
        - **点击场景家具**：系统检测点击位置是否命中家具碰撞体。若命中，触发该家具对应的预设交互（如点击床触发“躺下”动画，点击冰箱触发“拿饮料”动画）。
        - **点击角色身体**：系统检测点击位置是否命中角色身体碰撞体（细分头部、胸部、腰部、腿部、手部）。若命中，触发对应部位的触摸反馈。
    4. **迷你游戏**：若玩家已购买对应迷你游戏包，宿舍内会出现对应的游戏道具（如：一个棋盘、一个抓娃娃机）。点击道具，进入小游戏界面。
    5. **拍照模式**：点击屏幕右上角的相机图标，进入拍照模式。在该模式下，所有交互按钮隐藏，玩家可调整镜头、滤镜、角色姿势（需购买姿势包）。
- **【数据状态变化】**：
    - **交互触发**：无数据变化。所有交互均为纯表现层反馈。
    - **迷你游戏**：游戏结果（胜/负）记录在本地缓存，用于触发胜利/失败动画，不产生任何全局数据变化。
    - **拍照模式**：截图保存至本地相册，服务器不存储。
- **【表现层反馈】**：
    - **交互反馈**：每次交互触发对应的角色动作、语音、表情变化及物理抖动反馈。反馈具有0.5秒的冷却时间，防止玩家快速点击导致表现错乱。
    - **迷你游戏**：进入小游戏时，镜头自动切换至俯视或平视特写，UI风格变为轻快明亮的卡通风。
    - **拍照模式**：进入拍照模式时，屏幕边缘出现半透明取景框，底部出现滤镜选择轮盘。
- **【边界与异常兜底】**：
    - **无冷却时间**：交互与游戏无每日次数限制，可反复触发。但为防止性能问题，单次交互动画播放期间，新的交互指令将被缓存，待当前动画结束后执行。
    - **迷你游戏中途退出**：玩家在小游戏中点击“退出”按钮，系统弹窗确认“确定要离开吗？当前游戏进度不会保存。”，确认后返回宿舍主视角。
    - **拍照模式内存溢出**：若设备内存不足，拍照模式自动禁用滤镜与高分辨率选项，并提示“当前设备性能不足，已为您切换至基础拍照模式”。

### 表现层与角色展示联动
- **【前置条件】**：玩家已进入角色宿舍。
- **【流转逻辑】**：
    1. **UI/场景设计**：
        - 场景加载完成后，系统强制锁定为第一人称近景视角，角色模型占据屏幕中央60%-70%面积。
        - 所有UI按钮（拍照、游戏入口、设置）采用半透明浮动式，默认透明度为30%，当玩家停止操作3秒后，UI自动渐隐至完全透明，再次点击屏幕任意位置恢复显示。
    2. **擦边互动设计**：
        - **触摸反馈**：点击角色不同部位，系统根据`dorm_interaction_matrix`表查询对应的动画、语音、表情ID，并执行。
        - **物理引擎**：系统根据`device_quality_preset`表中的配置，决定是否启用软体物理与布料物理。
        - **语音系统**：每次交互触发时，从`voice_pack_id`对应的语音包中随机选取一条播放。同一部位连续点击，不会重复播放同一条语音。
    3. **特殊解锁内容**：
        - 玩家购买特定家具包后，宿舍场景中会出现该家具。点击该家具，系统检测是否拥有对应的`interaction_animation_id`，若有则播放专属互动动画。
        - 所有专属互动动画播放完毕后，自动进入慢镜头回放模式，持续5秒，方便玩家截图。
- **【数据状态变化】**：
    - **触摸反馈**：记录玩家点击部位与次数（仅用于本地统计，不上报）。
    - **专属动画**：播放完成后，在本地标记该动画已观看，下次点击同一家具时，跳过慢镜头回放，直接播放正常动画。
- **【表现层反馈】**：
    - **触摸反馈**：角色身体对应部位出现短暂的粉色光晕（持续0.3秒），随后触发动画与语音。
    - **专属动画**：播放时，屏幕上下出现电影级黑边，镜头自动切换至最佳观赏角度。
- **【边界与异常兜底】**：
    - **底线遵守**：所有触摸反馈动画均经过合规审查。若检测到玩家尝试通过修改客户端数据触发未定义交互，系统将强制断开连接并提示“检测到异常操作，已断开连接”。
    - **语音播放冲突**：若同时触发多个交互，系统建立语音播放队列，按触发顺序依次播放，不重叠。
    - **物理引擎性能回退**：若设备在开启物理效果后帧率持续低于30帧超过5秒，系统自动将物理效果降级（如：关闭软体物理，仅保留布料物理），并提示“检测到性能波动，已为您优化视觉效果”。

### 经济循环与商业化埋点
- **【前置条件】**：玩家已进入商城界面（宿舍内嵌商城）。
- **【流转逻辑】**：
    1. **付费道具分类**：商城界面分为“家具”、“姿势”、“服装”、“小游戏”四个标签页。每个道具以3D模型预览图展示，并附带“试穿/试用”按钮。
    2. **购买流程**：玩家点击道具，弹出购买确认窗，显示道具名称、价格、效果描述。点击“确认购买”，系统调用支付接口。
    3. **购买后处理**：支付成功后，道具立即解锁。若为家具/服装，宿舍场景/角色模型即时更新。若为姿势/小游戏，拍照模式/宿舍内出现新选项/道具。
    4. **免费内容**：每个角色默认解锁基础宿舍（一张床、一张桌子、一把椅子）与3个基础触摸互动（摸头、摸肩、牵手）。
    5. **付费刺激设计**：宿舍界面右上角常驻“推荐”按钮，点击后弹出轮播图，展示最新上架的道具，并自动播放角色试穿/试用动画预览。
- **【数据状态变化】**：
    - **购买成功**：玩家账户扣除对应金额，道具ID添加至玩家背包数据表。
    - **购买失败**：玩家账户金额不变，道具ID不添加，弹窗提示失败原因（余额不足/支付渠道错误）。
    - **每日奖励**：每日首次进入任意角色宿舍，系统调用`affection_bonus`接口，为主界面好感度系统增加少量数值。
- **【表现层反馈】**：
    - **购买成功**：屏幕中央出现“购买成功！”的闪光特效，持续2秒。角色会主动发送一条感谢短信（通过短信系统接口）。
    - **购买失败**：弹窗提示失败原因，并提供“重试”与“取消”按钮。
    - **每日奖励**：进入宿舍时，屏幕左上角出现“+XX好感度”的飘字提示。
- **【边界与异常兜底】**：
    - **支付中断**：若玩家在支付过程中断线或退出，系统在下次登录时检测支付状态。若支付成功但道具未发放，自动补发；若支付失败，不做处理。
    - **重复购买**：若玩家尝试购买已拥有的道具，系统弹窗提示“您已拥有该道具，无需重复购买”。
    - **短信发送失败**：若购买后调用短信系统接口失败，系统在本地记录失败日志，并在下次登录时重试发送。若连续3次失败，则放弃发送，不影响玩家体验。

### 旧系统与数据联动
- **【前置条件】**：玩家已进入宿舍系统。
- **【流转逻辑】**：
    1. **角色数据接口**：宿舍系统启动时，调用`get_character_list`接口获取玩家拥有的所有角色信息，包括`character_id`、`character_name`、`rarity`、`model_3d_path`、`voice_pack_id`。
    2. **好感度系统接口**：每日首次进入宿舍时，调用`add_affection_bonus(character_id, amount)`接口，增加对应角色的好感度。
    3. **拍照系统接口**：玩家在拍照模式下点击“截图”按钮，调用`take_screenshot(hide_ui=true, filter_id, resolution)`接口，生成并保存截图。
    4. **短信系统接口**：玩家购买道具后，调用`send_message(character_id, message_template_id)`接口，触发角色发送感谢短信。
- **【数据状态变化】**：
    - **角色数据接口**：无数据变化，仅读取。
    - **好感度系统接口**：对应角色的好感度数值增加。
    - **拍照系统接口**：无服务器数据变化，仅本地操作。
    - **短信系统接口**：在短信系统中生成一条新消息。
- **【表现层反馈】**：
    - **角色数据接口**：加载角色列表时，界面显示加载动画。
    - **好感度系统接口**：增加好感度时，界面出现飘字提示。
    - **拍照系统接口**：截图成功后，界面出现“已保存至相册”的提示。
    - **短信系统接口**：购买成功后，界面出现“角色发来一条新消息”的横幅提示。
- **【边界与异常兜底】**：
    - **接口调用失败**：若任一接口调用失败，系统记录错误日志，并尝试重试3次。若仍失败，则跳过该步骤，不影响宿舍系统核心功能。
    - **好感度接口限流**：若好感度接口有每日调用次数限制，系统在调用前检查本地缓存，确保每日仅调用一次。
    - **短信接口不存在**：若短信系统尚未开发完成，购买成功后仅显示UI提示“角色似乎想对你说些什么...”，不调用接口。

### 新增全局数据表
- **【前置条件】**：系统设计阶段。
- **【流转逻辑】**：
    1. **`dorm_furniture`表**：由数值策划填写，定义所有家具的ID、名称、价格、关联互动动画ID、模型路径、预览图路径。
    2. **`dorm_pose`表**：由数值策划填写，定义所有姿势的ID、名称、价格、模型骨骼配置、预览图路径。
    3. **`dorm_costume`表**：由数值策划填写，定义所有服装的ID、名称、价格、模型路径、语音包ID、预览图路径。
    4. **`dorm_minigame`表**：由战斗策划填写，定义所有小游戏的ID、名称、价格、游戏逻辑脚本路径、预览图路径。
    5. **`device_quality_preset`表**：由系统策划与程序共同维护，定义设备型号到画质等级的映射关系，以及各画质等级下的物理开关、模型LOD等级、阴影质量、抗锯齿等级等参数。
- **【数据状态变化】**：无。这些是静态配置表，在游戏版本更新时加载。
- **【表现层反馈】**：无。
- **【边界与异常兜底】**：
    - **配置表缺失**：若游戏加载时发现某张配置表缺失或格式错误，系统将使用硬编码的默认配置（如：所有画质设为“中”，所有道具价格设为0），并记录错误日志。
    - **设备型号未匹配**：若玩家设备型号未在`device_quality_preset`表中找到对应项，系统将根据设备CPU/GPU型号、内存大小进行模糊匹配，若仍无法匹配，则默认使用“中”画质。
    - **道具ID冲突**：若不同表中的道具ID发生冲突，系统在加载时弹窗提示“数据配置错误，请联系客服”，并阻止游戏进入宿舍系统。
