# 角色宿舍 - 系统详细设计案

## 一、系统概述与设计愿景

### 1.1 系统定位
- **核心定位**：高自由度角色展示与自定义空间，强调换装、拍照、场景布置，支持截图分享至社交平台
- **前置条件**：玩家必须拥有至少一名已获取的角色
- **数据归属**：宿舍场景配置、角色装扮状态、拍照数据均以玩家ID+角色ID为复合主键持久化
- **入口规则**：主界面“宿舍”入口常驻，点击后进入角色选择界面，再加载对应角色的宿舍场景
- **无产出定位**：本系统不产出任何战斗资源、抽卡资源或基础货币，所有付费内容均为一次性解锁

### 1.2 设计目标
- **核心目标**：为玩家提供“展示角色魅力”的创作与分享平台，强化角色厨力与社交传播
- **非目标**：不包含触摸互动、送礼、对话等ML向深度互动，不产出任何战斗资源
- **成功指标**：日均拍照次数、截图分享率、换装频率、场景布置修改频率

### 1.3 名词解释
- **方案**：包含场景模板ID、家具摆放数据、角色装扮ID、姿势ID、表情ID的完整配置集合
- **场景模板**：预设的宿舍基础环境（卧室、阳台、花园等），包含固定建筑结构与光照
- **家具/装饰品**：可自由摆放的3D物件，支持位置、旋转、缩放调整
- **皮肤/配件**：角色可更换的外观（服装、头饰、武器外观等）
- **姿势**：角色在场景中的预设动作状态（站立、坐姿、依靠等）
- **表情**：角色面部预设情绪表现（微笑、眨眼、害羞等）
- **滤镜**：拍照模式下叠加的画面色彩/光影效果
- **特效**：拍照模式下叠加的动态粒子/光效

## 二、核心规则与玩法机制

### 2.1 角色选择与场景加载
- **【前置条件】**：玩家已拥有至少一名角色
- **【流转逻辑】**
  1. 玩家点击主界面“宿舍”入口
  2. 系统弹出角色选择界面，展示玩家所有已获取角色的头像与名称
  3. 玩家选择一名角色后，系统加载该角色的专属宿舍场景
  4. 加载完成后，角色以当前保存的方案状态出现在场景中
- **【数据状态变化】**
  - `current_selected_character_id` 更新为所选角色ID
  - 加载该角色的 `character_dormitory_config` 数据（场景模板、家具、装扮、姿势、表情）
- **【表现层反馈】**
  - 角色选择界面以横向滑动列表展示所有角色头像
  - 加载过程中显示加载动画与进度条
  - 加载完成后，场景淡入，角色以当前姿势与表情出现在场景中心
- **【边界与异常兜底】**
  - 若玩家未拥有任何角色，入口按钮置灰并提示“请先获取一名角色”
  - 若角色场景配置数据损坏，自动重置为默认配置
  - 若加载超时（超过 `[SCENE_LOAD_TIMEOUT]` 秒），显示“加载失败，请重试”提示

### 2.2 换装系统
- **【前置条件】**：玩家拥有至少一件该角色可用的皮肤/配件
- **【流转逻辑】**
  1. 玩家点击换装按钮，进入换装界面
  2. 界面展示该角色所有已拥有的皮肤/配件列表，按部位分类（服装、头饰、武器外观等）
  3. 玩家点击任意皮肤/配件，角色模型实时预览切换效果
  4. 玩家点击“确认”按钮，保存当前装扮状态
  5. 玩家点击“取消”按钮，恢复至进入换装界面前的装扮状态
- **【数据状态变化】**
  - 确认后，`character_current_skin_id` 更新为所选皮肤ID
  - 若玩家切换了多个部位，每个部位的ID分别更新
  - 方案数据中的装扮部分同步更新
- **【表现层反馈】**
  - 换装界面以网格列表展示所有可用皮肤/配件，已装备项显示“已装备”标签
  - 点击预览时，角色模型立即切换，播放换装动画（如旋转展示）
  - 确认后，界面关闭，角色保持新装扮
- **【边界与异常兜底】**
  - 若玩家未拥有任何皮肤/配件，换装按钮置灰并提示“暂无可用装扮”
  - 若玩家在预览状态下退出换装界面，自动恢复至进入前的装扮状态
  - 若玩家同时切换多个部位，所有切换操作在确认后统一保存

### 2.3 姿势系统
- **【前置条件】**：玩家已解锁至少一个姿势
- **【流转逻辑】**
  1. 玩家点击姿势按钮，弹出姿势选择面板
  2. 面板展示该角色所有已解锁的姿势列表（站立、坐姿、依靠、蹲姿等）
  3. 玩家点击任意姿势，角色模型实时切换至该姿势
  4. 玩家点击确认或关闭面板，姿势状态保存
- **【数据状态变化】**
  - `character_current_pose_id` 更新为所选姿势ID
  - 方案数据中的姿势部分同步更新
- **【表现层反馈】**
  - 姿势选择面板以图标列表展示，当前姿势高亮显示
  - 切换姿势时，角色播放过渡动画（如从站立到坐下的坐下动画）
  - 未解锁的姿势显示锁图标与解锁条件
- **【边界与异常兜底】**
  - 若玩家仅拥有基础姿势，姿势选择面板仅显示基础姿势
  - 若玩家在角色处于特定姿势时切换场景模板，姿势自动适配新场景（如无对应适配，恢复为默认站立姿势）

### 2.4 表情系统
- **【前置条件】**：玩家已解锁至少一个表情
- **【流转逻辑】**
  1. 玩家点击表情按钮，弹出表情选择面板
  2. 面板展示该角色所有已解锁的表情列表（微笑、眨眼、害羞、严肃等）
  3. 玩家点击任意表情，角色面部实时切换至该表情
  4. 玩家点击确认或关闭面板，表情状态保存
- **【数据状态变化】**
  - `character_current_expression_id` 更新为所选表情ID
  - 方案数据中的表情部分同步更新
- **【表现层反馈】**
  - 表情选择面板以图标列表展示，当前表情高亮显示
  - 切换表情时，角色面部平滑过渡
  - 未解锁的表情显示锁图标与解锁条件
- **【边界与异常兜底】**
  - 若玩家仅拥有基础表情，表情选择面板仅显示基础表情
  - 若玩家在拍照模式下切换表情，表情切换不影响拍照模式状态

### 2.5 场景布置
- **【前置条件】**：玩家已解锁至少一个场景模板
- **【流转逻辑】**
  1. 玩家点击场景布置按钮，进入布置模式
  2. 布置模式下，界面底部显示家具/装饰品分类列表（椅子、桌子、花束、灯光、背景板等）
  3. 玩家从列表中选择一个家具/装饰品，该物件出现在场景中，跟随玩家手指/鼠标移动
  4. 玩家点击场景中的目标位置，物件放置在该位置
  5. 玩家可选中已放置的物件，进行位置微调、旋转（每次 `[ROTATION_STEP]` 度）、缩放（每次 `[SCALE_STEP]` 倍）
  6. 玩家点击“保存”按钮，当前布置状态保存
  7. 玩家点击“退出”按钮，退出布置模式
- **【数据状态变化】**
  - `scene_furniture_data` 更新为当前所有物件的ID、位置坐标、旋转角度、缩放比例
  - 方案数据中的场景布置部分同步更新
- **【表现层反馈】**
  - 布置模式下，场景中所有可交互物件显示半透明选中框
  - 玩家拖动物件时，物件跟随移动，场景中显示放置预览（绿色表示可放置，红色表示不可放置）
  - 旋转/缩放操作时，物件实时变化
  - 保存后，场景中物件固定，退出布置模式
- **【边界与异常兜底】**
  - 物件放置时进行碰撞检测，若与场景中其他物件或角色重叠，显示红色提示并禁止放置
  - 单个场景中可放置的物件数量上限为 `[MAX_FURNITURE_COUNT]`
  - 若玩家在布置模式下切换场景模板，所有未保存的布置操作丢失，系统提示“切换场景将丢失未保存的布置，是否继续？”
  - 若玩家在布置模式下断线，所有未保存的布置操作丢失

### 2.6 场景模板切换
- **【前置条件】**：玩家已解锁至少两个场景模板
- **【流转逻辑】**
  1. 玩家点击场景模板切换按钮，弹出模板选择面板
  2. 面板展示所有已解锁的场景模板列表（卧室、阳台、花园、泳池边等）
  3. 玩家点击任意模板，场景实时切换至该模板
  4. 切换后，原场景中的家具/装饰品自动适配新模板（若新模板有对应位置则保留，否则自动回收至背包）
  5. 玩家点击确认，切换生效
- **【数据状态变化】**
  - `scene_template_id` 更新为所选模板ID
  - 家具数据中无法适配的物件ID从场景中移除，标记为“背包中”
  - 方案数据中的场景模板部分同步更新
- **【表现层反馈】**
  - 模板选择面板以缩略图展示，当前模板高亮显示
  - 切换时，场景播放过渡动画（如淡入淡出）
  - 无法适配的家具显示“已回收至背包”提示
- **【边界与异常兜底】**
  - 若玩家仅拥有一个场景模板，模板切换按钮置灰
  - 若玩家在切换模板时未保存当前布置，系统自动保存当前布置后再切换

### 2.7 拍照模式
- **【前置条件】**：玩家处于宿舍场景中
- **【流转逻辑】**
  1. 玩家点击拍照按钮，进入拍照模式
  2. 进入后，所有UI自动隐藏，仅保留角色与场景
  3. 玩家可自由旋转/缩放/平移镜头（触屏双指缩放/旋转，单指平移）
  4. 玩家点击滤镜按钮，弹出滤镜选择面板，选择后画面实时叠加滤镜效果
  5. 玩家点击特效按钮，弹出特效选择面板，选择后画面实时叠加特效效果
  6. 玩家可点击角色调整按钮，微调角色位置、朝向、姿势、表情
  7. 玩家点击快门按钮，系统截取当前画面
  8. 截图后，弹出分享面板，玩家可选择保存至本地相册或一键分享至社交平台
  9. 玩家点击退出按钮，退出拍照模式，恢复UI显示
- **【数据状态变化】**
  - 拍照过程中无后台数据变化
  - 截图保存至本地相册，文件名为 `[CHARACTER_NAME]_[TIMESTAMP].png`
  - 分享图默认包含游戏水印（水印位置为右下角，透明度 `[WATERMARK_OPACITY]`）
- **【表现层反馈】**
  - 进入拍照模式后，所有UI淡出，画面全屏显示角色与场景
  - 滤镜/特效选择面板以半透明毛玻璃样式展示，不遮挡画面主体
  - 快门按钮按下时，播放快门音效与闪光动画
  - 截图完成后，显示“已保存至相册”提示
  - 分享面板显示社交平台图标列表（微信、微博、QQ等）
- **【边界与异常兜底】**
  - 若玩家在拍照模式下切换角色，自动退出拍照模式并返回角色选择界面
  - 若玩家在拍照模式下断线，拍照模式正常使用，但分享功能不可用
  - 若设备存储空间不足，截图保存失败，提示“存储空间不足，请清理后重试”
  - 分享功能调用系统API，若目标社交平台未安装，提示“请先安装该应用”

### 2.8 方案保存与切换
- **【前置条件】**：玩家已保存至少一个方案
- **【流转逻辑】**
  1. 玩家点击方案保存按钮，弹出命名输入框
  2. 玩家输入方案名称（最多 `[SCHEME_NAME_MAX_LENGTH]` 个字符），点击确认
  3. 系统将当前场景模板、家具数据、角色装扮、姿势、表情保存为一个方案
  4. 方案保存后，在方案列表中显示
  5. 玩家点击方案切换按钮，弹出方案列表
  6. 玩家点击任意方案，场景与角色立即切换至该方案状态
- **【数据状态变化】**
  - 保存时，`character_scheme_data` 新增一条记录，包含方案ID、名称、场景模板ID、家具数据、装扮ID、姿势ID、表情ID
  - 切换时，`current_scheme_id` 更新为所选方案ID，场景与角色数据同步更新
- **【表现层反馈】**
  - 方案保存成功后，显示“方案已保存”提示
  - 方案列表以缩略图展示（缩略图为保存时自动截取的场景预览图）
  - 切换方案时，场景与角色播放过渡动画
- **【边界与异常兜底】**
  - 免费提供 `[FREE_SCHEME_SLOT_COUNT]` 个方案槽位，超出需付费解锁
  - 若方案槽位已满，保存按钮置灰并提示“方案槽位已满，请先删除或付费解锁”
  - 玩家可删除已保存的方案，删除后槽位释放
  - 若玩家在切换方案时未保存当前状态，系统自动保存当前状态为临时方案（不占用槽位）

## 三、表现层与角色展示联动

### 3.1 运镜设计
- **默认视角**：进入宿舍后，默认采用“第三人称近肩视角”，镜头略微仰视，角色始终处于画面中心
- **拍照模式**：完全自由镜头，无任何UI遮挡，支持触屏双指缩放/旋转，单指平移
- **场景浏览**：玩家可自由拖动视角浏览整个场景，无锁定
- **视角重置**：玩家可点击“重置视角”按钮，镜头恢复至默认视角

### 3.2 物理反馈
- **角色物理**：角色模型搭载头发、裙摆、胸部物理抖动，换装后新模型同样适配物理
- **场景物理**：部分装饰品（如窗帘、吊灯）有轻微物理动画，增强沉浸感
- **物理分级**：提供“高/中/低”三档物理效果选项，低档关闭所有物理动画，中档保留头发与裙摆物理，高档保留全部物理

### 3.3 UI透明化
- **默认状态**：所有UI组件（功能按钮、角色信息）采用半透明毛玻璃设计，不遮挡角色主体
- **一键隐藏**：玩家可点击“隐藏UI”按钮一键隐藏所有UI，仅保留角色与场景
- **拍照模式**：自动隐藏所有UI，退出后恢复
- **UI透明度**：玩家可在设置中调整UI透明度（`[UI_OPACITY_MIN]` 至 `[UI_OPACITY_MAX]`）

### 3.4 擦边底线
- **姿势与表情**：所有姿势与表情严格控制在“展示魅力”范畴，不包含任何性暗示动作或台词
- **换装尺度**：皮肤/配件设计遵循项目宪法红线，严禁露点或直接性行为暗示
- **拍照分享**：截图分享功能不涉及任何违规内容审核，但游戏内不提供社交互动功能（如评论、点赞）
- **物理抖动**：物理抖动仅限头发、裙摆、胸部等常规部位，不包含任何敏感部位的特写或放大

## 四、经济循环与商业化埋点

### 4.1 免费产出
- **基础场景模板**：初始提供 `[FREE_SCENE_TEMPLATE_COUNT]` 个免费场景模板
- **基础姿势与表情**：每个角色初始提供 `[BASE_POSE_COUNT]` 个基础姿势与 `[BASE_EXPRESSION_COUNT]` 个基础表情
- **基础滤镜**：初始提供 `[FREE_FILTER_COUNT]` 款免费滤镜
- **方案槽位**：免费提供 `[FREE_SCHEME_SLOT_COUNT]` 个方案槽位

### 4.2 付费埋点
- **皮肤/配件**：高质量皮肤与配件为主要付费点，部分皮肤可单独购买，部分随抽卡/活动获取
- **场景模板**：高级场景模板（如“豪华泳池”、“星空露台”）为付费解锁
- **家具/装饰品**：特殊家具与装饰品（如“动态樱花树”、“发光魔法阵”）为付费购买
- **滤镜与特效**：高级滤镜与特效（如“赛博朋克”、“水墨风”）为付费解锁
- **姿势与表情**：特殊姿势与表情（如“躺姿”、“眨眼wink”）为付费解锁
- **方案槽位**：额外方案槽位需付费解锁

### 4.3 资源循环
- **无产出**：宿舍系统不产出任何战斗资源、抽卡资源或基础货币
- **纯消费**：所有付费内容均为一次性解锁，无消耗性道具
- **购买方式**：所有付费内容通过商城系统购买，购买后永久解锁

## 五、旧系统与数据联动

### 5.1 角色获取系统
- **联动规则**：玩家通过抽卡或活动获得新角色后，自动解锁该角色的基础宿舍场景与基础姿势/表情
- **数据同步**：角色ID同步至宿舍系统，作为宿舍场景的角色主体
- **异常兜底**：若玩家重复获得已拥有角色，不重复解锁宿舍内容

### 5.2 商城系统
- **联动规则**：商城系统新增“宿舍”分类，售卖皮肤/配件、场景模板、家具/装饰品、滤镜/特效、姿势/表情、方案槽位
- **数据同步**：购买后道具自动进入玩家背包，宿舍系统读取背包数据展示可用内容
- **异常兜底**：若玩家购买后背包数据未及时同步，宿舍系统在下次进入时自动刷新

### 5.3 图鉴系统
- **联动规则**：宿舍系统中解锁的姿势、表情、场景模板等自动同步至图鉴系统，玩家可在图鉴中回看已解锁内容
- **数据同步**：图鉴系统读取宿舍系统的解锁状态，展示对应内容
- **异常兜底**：若宿舍系统解锁状态与图鉴系统不一致，以宿舍系统数据为准

### 5.4 活动系统
- **联动规则**：限时活动可产出宿舍专属道具（如限定场景模板、限定家具、限定姿势）
- **数据同步**：活动奖励发放至玩家背包，宿舍系统读取后可用
- **异常兜底**：若活动结束后玩家未领取奖励，奖励通过邮件补发

### 5.5 画质分级系统
- **联动规则**：宿舍系统提供“低/中/高”三档画质选项，低档关闭物理效果与阴影，中档保留基础物理与阴影，高档保留全部效果
- **数据同步**：画质设置保存至玩家全局配置，宿舍系统读取后应用
- **异常兜底**：若设备性能不足，系统自动推荐中档画质
