# 角色私人宿舍 - 系统详细设计案

### [角色房间系统]
- **【前置条件】**：
  - 玩家已在主游戏中通过抽卡或活动获得该角色（强依赖）。
  - 玩家已购买“角色私人宿舍”DLC（一次性买断）。
  - 玩家从主界面“私人宿舍”入口进入，选择已拥有的角色。
- **【流转逻辑】**：
  1. 玩家点击“私人宿舍”入口，系统加载角色选择界面。
  2. 玩家从已拥有角色列表中选择一个角色。
  3. 系统加载该角色的专属3D房间（包含所有已解锁的家具、摆件、纪念品）。
  4. 角色以AI日常行为状态出现在房间内（如坐在沙发、站在窗边）。
  5. 玩家可自由旋转/缩放视角（触屏双指缩放/旋转，PC鼠标滚轮+右键拖拽）。
  6. 玩家点击房间内热点（沙发、床、书桌、窗户、纪念品展示架等）触发互动。
  7. 玩家可随时点击“退出”按钮返回角色选择界面或主界面。
- **【数据状态变化】**：
  - 加载房间时，读取该角色的 `room_config`（家具摆放、纪念品位置、装饰颜色方案）。
  - 加载角色时，读取该角色的 `current_outfit_id`（当前穿戴皮肤），应用对应模型与材质。
  - 加载互动热点时，读取 `hotspot_status`（每个热点是否已被触发过、当前可用状态）。
- **【表现层反馈】**：
  - 房间加载时，展示淡入动画（从门缝视角推入房间）。
  - 角色出现时，播放一个简短的“欢迎回家”动作（如转身微笑、挥手）。
  - 视角控制时，UI显示极简的“视角锁定”图标（防止误触退出）。
  - 退出时，播放角色挥手告别的动画与语音。
- **【边界与异常兜底】**：
  - 若玩家选择角色时网络中断，显示“数据加载失败，请检查网络”弹窗，返回主界面。
  - 若角色模型加载超时（>10秒），显示“资源加载中...”进度条，超时后降级为低精度模型（LOD0）。
  - 若房间配置数据损坏，自动恢复为默认房间布局，并记录错误日志。

### [互动热点系统]
- **【前置条件】**：
  - 角色房间已成功加载，角色处于AI日常行为状态。
  - 热点未被锁定（如好感度未达到解锁条件的热点显示为灰色）。
  - 玩家视角未处于拍照模式。
- **【流转逻辑】**：
  1. 玩家视角靠近热点（距离<3米）时，热点高亮并显示交互图标（如“手”图标）。
  2. 玩家点击热点，系统判定当前热点类型（沙发、床、书桌、窗户、纪念品）。
  3. 系统读取该热点对应的互动动画ID列表（根据好感度等级不同，动画可能不同）。
  4. 系统触发角色从当前AI状态切换到互动状态（打断当前行为，播放过渡动画）。
  5. 播放互动动画（强制第一人称或近景特写运镜），同时播放对应语音。
  6. 互动动画播放完毕后，角色返回AI日常行为状态。
  7. 系统增加好感度经验值（根据热点类型和好感度等级计算）。
- **【数据状态变化】**：
  - 热点状态 `hotspot_status` 更新为“已触发”，记录触发时间戳。
  - 好感度经验值 `affection_exp` 增加（如沙发互动+10exp，床互动+15exp）。
  - 若热点有每日次数限制，更新 `daily_interaction_count`。
  - 若互动解锁了新的对话或动画，更新 `unlocked_dialogues` 列表。
- **【表现层反馈】**：
  - 热点高亮时，显示半透明光晕和交互图标（颜色根据热点类型不同）。
  - 互动开始时，UI完全隐藏（仅保留极小的“跳过”按钮在屏幕角落）。
  - 互动动画播放时，镜头强制切换为预设运镜（如仰视、俯视、特写）。
  - 互动结束后，显示好感度增加的气泡提示（如“好感度+10”），气泡以手机消息样式弹出。
  - 若解锁新对话，显示“新消息”图标在屏幕边缘，点击可查看。
- **【边界与异常兜底】**：
  - 若玩家在互动动画播放中点击“跳过”，立即结束动画，角色返回AI状态，好感度奖励照常发放。
  - 若热点每日次数耗尽（如每日限3次），热点显示为灰色，点击提示“今天已经互动过了，明天再来吧”。
  - 若好感度未达到解锁条件，热点显示为锁定状态（锁图标），点击提示“好感度达到X级后解锁”。
  - 若互动动画文件缺失，播放默认的“微笑点头”动画，并记录错误日志。

### [小游戏系统]
- **【前置条件】**：
  - 角色房间已成功加载。
  - 玩家已解锁该角色的小游戏入口（好感度达到1级自动解锁第一个小游戏）。
  - 小游戏未处于冷却时间（每个小游戏每日可挑战3次）。
- **【流转逻辑】**：
  1. 玩家点击房间内的小游戏入口（如书桌上的“游戏机”、墙上的“飞镖盘”）。
  2. 系统加载小游戏场景（切换至独立游戏界面，房间背景模糊）。
  3. 玩家进行小游戏（如换装搭配、料理制作、射击打靶、节奏音游）。
  4. 小游戏结束，系统判定胜负/得分。
  5. 根据结果发放奖励（好感度经验、纪念品碎片、剧情解锁进度）。
  6. 玩家点击“返回房间”按钮，系统卸载小游戏场景，返回角色房间。
- **【数据状态变化】**：
  - 小游戏每日挑战次数 `daily_minigame_count` 减1。
  - 好感度经验值 `affection_exp` 增加（胜利+30exp，失败+10exp）。
  - 若小游戏有成就系统，更新 `minigame_achievements`（如“累计得分超过1000”）。
  - 若小游戏产出纪念品碎片，更新 `souvenir_fragment_count`。
- **【表现层反馈】**：
  - 小游戏入口显示为可交互的3D物体（如发光的游戏机、旋转的飞镖盘）。
  - 进入小游戏时，屏幕淡入至游戏界面，显示游戏名称和操作提示。
  - 游戏过程中，UI显示当前得分、剩余时间/次数、连击数（如适用）。
  - 游戏结束时，显示结果弹窗（胜利/失败、得分、奖励列表）。
  - 返回房间时，角色会有一个“等你回来”的动作（如放下手中的书，看向玩家）。
- **【边界与异常兜底】**：
  - 若每日挑战次数耗尽，小游戏入口显示为灰色，点击提示“今日挑战次数已用完”。
  - 若玩家在小游戏中断线，游戏暂停并显示“网络连接中断，请重连”，重连后继续游戏（限时30秒）。
  - 若小游戏加载超时（>15秒），显示“加载失败，请重试”，返回房间。
  - 若小游戏得分数据异常（如负数），强制判定为失败，并记录错误日志。

### [好感度系统]
- **【前置条件】**：
  - 角色房间已成功加载。
  - 好感度系统独立于主游戏好感度，初始等级为0级。
- **【流转逻辑】**：
  1. 玩家通过互动热点或小游戏获得好感度经验值 `affection_exp`。
  2. 系统累计经验值，当达到升级阈值时，触发升级事件。
  3. 升级后，解锁对应等级的内容（新互动动画、新对话、新小游戏、房间装饰变化）。
  4. 好感度等级上限为10级（首发版本）。
  5. 好感度经验值不会衰减或重置。
- **【数据状态变化】**：
  - `affection_exp` 增加（每次互动/小游戏后）。
  - `affection_level` 在达到阈值时更新（如0→1需100exp，1→2需300exp）。
  - `unlocked_content` 列表更新（新增解锁的动画ID、对话ID、小游戏ID、装饰ID）。
  - 若达到特定等级（如5级、10级），触发主游戏联动奖励（解锁头像框或名片背景）。
- **【表现层反馈】**：
  - 好感度经验值增加时，显示手机消息样式的气泡（如“好感度+10”）。
  - 升级时，屏幕中央显示“好感度提升！”动画（心形特效+角色特写）。
  - 解锁新内容时，显示“新内容解锁！”提示，并引导玩家查看（如“新的互动动作已解锁，去沙发看看吧”）。
  - 好感度等级显示在房间UI的角落（极简的爱心图标+数字）。
  - 达到联动等级时，显示“主游戏奖励已解锁！”弹窗。
- **【边界与异常兜底】**：
  - 若经验值溢出（如满级后继续获得经验），经验值不再累积，但正常显示“好感度已满”提示。
  - 若升级阈值数据错误（如负数），强制锁定等级为0，并记录错误日志。
  - 若联动奖励发放失败（如主游戏数据异常），显示“奖励发放失败，请联系客服”，并记录错误日志。

### [剧情解锁系统]
- **【前置条件】**：
  - 好感度达到特定等级（如3级、6级、9级）或完成特定小游戏成就。
  - 剧情片段未被解锁过。
- **【流转逻辑】**：
  1. 玩家达到解锁条件后，系统弹出“新剧情片段已解锁”提示。
  2. 玩家点击提示或从房间内的“相册”入口进入剧情播放界面。
  3. 系统加载剧情片段（Live2D或3D演出，包含对话、动画、选项）。
  4. 剧情播放完毕后，系统标记该片段为“已观看”。
  5. 玩家可随时在“档案”系统中回顾已解锁的剧情片段（弱联动）。
- **【数据状态变化】**：
  - `unlocked_story_segments` 列表新增该片段ID。
  - `story_segment_status` 更新为“已解锁”。
  - 若剧情包含选项，记录玩家选择（`story_choices`），影响后续剧情解锁（如有）。
- **【表现层反馈】**：
  - 解锁提示以“手机消息”样式弹出（如“收到一条新消息：回忆片段已解锁”）。
  - 剧情播放时，UI完全隐藏（仅保留“跳过”和“自动播放”按钮）。
  - 剧情结束后，显示“回忆已保存”提示。
  - 在“档案”系统中，已解锁的剧情片段显示为彩色，未解锁的显示为灰色锁图标。
- **【边界与异常兜底】**：
  - 若剧情文件加载失败，显示“剧情加载失败，请重试”，返回房间。
  - 若玩家在剧情播放中退出，进度自动保存，下次进入可继续观看。
  - 若剧情选项数据异常（如选项ID不存在），强制跳过选项，使用默认剧情分支。

### [纪念品系统]
- **【前置条件】**：
  - 玩家已完成特定小游戏成就（如“在射击小游戏中累计得分超过5000”）或好感度达到6级。
  - 纪念品未被解锁过。
- **【流转逻辑】**：
  1. 玩家达成解锁条件后，系统判定纪念品解锁。
  2. 系统检查该纪念品是否需要碎片合成（`souvenir_fragment_count` 是否达到合成阈值）。
  3. 若需要碎片合成，系统检查 `souvenir_fragment_count` 是否 >= `fragment_required`（如10个碎片）。
  4. 若碎片数量足够，系统自动合成纪念品，并扣除对应碎片数量。
  5. 若碎片数量不足，系统显示“碎片进度”提示（如“碎片收集进度：7/10”）。
  6. 合成成功后，纪念品自动摆放在房间内的纪念品展示架上。
  7. 玩家可点击纪念品查看详情（名称、描述、解锁条件）。
- **【数据状态变化】**：
  - `souvenir_fragment_count` 减少（合成时扣除）。
  - `unlocked_souvenirs` 列表新增该纪念品ID。
  - `room_decorations` 列表新增该纪念品的摆放位置数据。
  - 若纪念品有特殊互动效果（如点击播放动画），更新 `hotspot_status`。
- **【表现层反馈】**：
  - 解锁条件达成时，显示“新纪念品可制作！”提示。
  - 碎片收集进度以“进度条”形式显示在纪念品详情界面。
  - 合成成功时，播放“合成成功”动画（闪光特效+纪念品特写）。
  - 纪念品摆放在展示架上后，玩家可点击查看，显示3D模型和文字描述。
  - 若碎片不足，显示“还差X个碎片”的提示，并引导玩家去玩对应小游戏。
- **【边界与异常兜底】**：
  - 若碎片数量足够但合成失败（如数据写入错误），显示“合成失败，请重试”，不扣除碎片。
  - 若纪念品摆放位置已被占用，自动调整到下一个可用位置。
  - 若纪念品ID不存在，显示“纪念品数据异常”，并记录错误日志。
  - 若玩家通过好感度6级解锁纪念品（无需碎片），直接跳过碎片检查，自动解锁并摆放。

### [AI日常行为系统]
- **【前置条件】**：
  - 角色房间已成功加载。
  - 角色未处于互动动画或小游戏状态。
- **【流转逻辑】**：
  1. 角色在房间内自动执行预设行为（如看书、发呆、做家务、哼歌）。
  2. 行为切换由状态机控制，基于时间间隔和随机概率。
  3. 每个行为有对应的动画、语音（随机触发）和移动路径。
  4. 玩家可随时点击角色触发互动，打断当前AI行为。
  5. 互动结束后，角色从AI状态机中恢复（从“空闲”状态重新开始）。
- **【数据状态变化】**：
  - `ai_behavior_state` 更新（如“reading”、“idle”、“cleaning”）。
  - `ai_behavior_timer` 重置（每个行为有最小持续时间，如5秒）。
  - `random_voice_timer` 更新（语音触发有冷却时间，如30秒）。
- **【表现层反馈】**：
  - 角色执行行为时，播放对应动画（如翻书、擦桌子、伸懒腰）。
  - 角色移动时，播放行走动画，路径平滑（避开家具碰撞）。
  - 语音触发时，显示对话气泡（如“今天天气真好呢~”）。
  - 角色发呆时，会有小动作（如拨弄头发、看窗外）。
- **【边界与异常兜底】**：
  - 若AI行为动画文件缺失，角色保持“空闲”状态（站立或坐下）。
  - 若角色移动路径被玩家视角阻挡，角色自动绕行或等待。
  - 若玩家长时间不互动（>5分钟），角色会触发“呼唤玩家”行为（如“主人，你在吗？”）。
  - 若AI状态机数据错误，强制重置为“空闲”状态，并记录错误日志。

### [拍照模式]
- **【前置条件】**：
  - 角色房间已成功加载。
  - 角色处于AI日常行为状态或互动动画状态（拍照模式可随时开启）。
- **【流转逻辑】**：
  1. 玩家点击屏幕角落的“相机”图标，进入拍照模式。
  2. 玩家可自由调整视角（旋转、缩放、平移）。
  3. 玩家可选择滤镜（如“暖色调”、“冷色调”、“黑白”）。
  4. 玩家可切换角色姿势（如“站立”、“坐下”、“躺下”）和表情（如“微笑”、“害羞”、“惊讶”）。
  5. 玩家点击“拍照”按钮，系统截图并保存至本地相册。
  6. 玩家点击“退出”按钮，返回正常互动模式。
- **【数据状态变化】**：
  - 拍照模式开启时，`photo_mode_active` 设为 true。
  - 角色姿势和表情切换时，更新 `character_pose` 和 `character_expression`。
  - 截图保存时，生成图片文件，记录截图时间戳。
- **【表现层反馈】**：
  - 进入拍照模式时，UI切换为拍照模式专属UI（滤镜选择、姿势切换、截图按钮）。
  - 视角控制时，显示网格辅助线（可选）。
  - 切换滤镜时，屏幕实时预览滤镜效果。
  - 切换姿势/表情时，角色立即切换对应动画/BlendShape。
  - 截图时，播放“咔嚓”音效和闪光灯效果。
  - 截图保存后，显示“已保存至相册”提示。
- **【边界与异常兜底】**：
  - 若玩家在拍照模式中点击互动热点，自动退出拍照模式并触发互动。
  - 若截图保存失败（如存储空间不足），显示“保存失败，请清理存储空间”。
  - 若角色姿势/表情数据不存在，使用默认姿势/表情。
  - 若玩家在拍照模式中断线，自动退出拍照模式，返回房间。
