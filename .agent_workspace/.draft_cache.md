# 游戏设计文档草案：角色私人宿舍系统

## 一、系统概述与设计愿景
- **系统目的**：为玩家提供一个与已拥有角色进行深度私密互动的专属空间，通过触摸、滑动等操作触发角色特定动作、语音及小游戏，强化“ML向情感羁绊”与“极致表现层”的核心支柱。
- **契合愿景**：
  - **角色展示**：系统以全3D角色模型为核心，所有交互均围绕角色身体与表情展开，无UI遮挡。
  - **情感羁绊**：互动结果直接影响角色好感度，解锁更亲密的行为与语音，营造专属陪伴感。
  - **擦边尺度**：所有互动严格控制在“性暗示”与“暧昧氛围”层面，无任何实质性行为或露点内容。

## 二、核心规则与玩法机制
### 2.1 系统入口与解锁
- **入口**：主界面“基地”按钮 → “私人宿舍”分页。
- **解锁条件**：玩家拥有该角色（抽卡获得）且角色等级达到20级。
- **购买制**：每个角色的“私人宿舍”为一次性买断（例如：648付费钻），购买后永久解锁该角色的所有基础互动内容。

### 2.2 互动模式
- **模式选择**：进入宿舍后，玩家可选择“自由互动”或“主题小游戏”。
  - **自由互动**：玩家可自由点击/滑动角色身体部位（头、胸、腰、臀、腿、手），触发预设动作与语音。
  - **主题小游戏**：预设3-5个固定小游戏（如“挠痒痒”、“喂食”、“换装”、“哄睡”），每个小游戏有独立规则与结算。

### 2.3 自由互动规则
- **[触发动作]**：玩家点击角色身体某部位（如胸部）。
  - **[判定条件]**：点击位置在预设碰撞区域内。
  - **✅ [单次成功反馈]**：
    - **表现层**：角色身体对应部位产生轻微抖动/挤压动画，角色触发随机语音（如“讨厌~”、“别这样…”），表情变为害羞/微怒。
    - **数值层**：好感度 +1。
  - **❌ [单次失败反馈]**：
    - **表现层**：角色无反应或触发拒绝语音（如“不行…”），表情变为冷淡。
    - **数值层**：无变化。
- **[触发动作]**：玩家在角色身体上滑动（如从腰滑到臀）。
  - **[判定条件]**：滑动轨迹覆盖至少两个不同部位碰撞区。
  - **✅ [单次成功反馈]**：
    - **表现层**：角色身体跟随滑动轨迹产生连续抖动，触发特殊语音（如“啊…那里…”），角色身体微微后仰。
    - **数值层**：好感度 +2。
  - **❌ [单次失败反馈]**：
    - **表现层**：角色身体僵硬，触发警告语音（如“请住手”）。
    - **数值层**：无变化。

### 2.4 主题小游戏规则（示例：“哄睡”）
- **玩法描述**：玩家需通过轻柔的抚摸与摇晃，让角色在床上入睡。
- **界面**：角色侧躺在床上，玩家可通过滑动屏幕进行抚摸。
- **规则**：
  - **[触发动作]**：玩家在角色背部/头部区域进行缓慢滑动。
    - **[判定条件]**：滑动速度低于阈值（如0.5秒/次）。
    - **✅ [单次成功反馈]**：
      - **表现层**：角色呼吸变缓，发出轻微鼾声，表情放松。
      - **数值层**：睡眠进度条 +5%。
    - **❌ [单次失败反馈]**：
      - **表现层**：角色皱眉，身体翻动，发出不满哼声。
      - **数值层**：睡眠进度条 -2%。
  - **[结算条件]**：睡眠进度条达到100%。
    - **✅ [胜利反馈]**：
      - **表现层**：播放角色熟睡特写动画（镜头缓慢推进，聚焦角色安详睡脸），触发专属语音（如“晚安，分析员…”）。
      - **数值层**：好感度 +50，获得道具“安眠香薰”（用于其他系统）。
    - **❌ [失败条件]**：玩家连续5次失败或主动退出。
      - **表现层**：角色坐起，揉眼睛，抱怨“睡不着…”。
      - **数值层**：好感度 -10。

## 三、表现层与角色展示联动 (核心强制项)
### 3.1 角色展示最大化
- **全屏3D模型**：宿舍场景中，角色模型占据屏幕60%以上面积，背景（床、沙发等）仅作为辅助构图元素。
- **动态镜头**：
  - **自由互动**：镜头自动跟随点击/滑动位置，并带有轻微呼吸感晃动。
  - **小游戏**：根据游戏类型预设固定机位（如“哄睡”为俯拍45度角，“换装”为正面全身）。
- **无UI遮挡**：互动过程中，所有UI元素（如好感度进度条）以半透明形式悬浮于屏幕边缘，核心区域始终留给角色。

### 3.2 擦边互动设计
- **物理反馈**：所有触摸互动均触发对应部位的软体物理抖动（Jiggle Physics），抖动幅度与力度成正比。
- **语音系统**：每个角色预设50+条互动语音，涵盖害羞、抗拒、愉悦、撒娇等情绪，按好感度阶段解锁。
- **特殊动作**：部分互动可触发角色特定动作（如被摸头时蹭手、被摸腰时扭动）。
- **底线遵守**：所有互动止步于“抚摸”、“滑动”、“摇晃”，无任何脱衣、性行为或露点内容。角色反应以“害羞”、“抗拒”、“愉悦”为主，不出现“高潮”等明确性暗示词汇。

## 四、经济循环与商业化埋点
### 4.1 付费点
- **买断制**：每个角色宿舍售价648付费钻（约合人民币648元），一次性解锁所有基础互动与小游戏。
- **额外内容**：后续可推出“互动扩展包”（如新小游戏、新语音包、新动作），售价198付费钻。

### 4.2 产出与消耗
- **产出**：
  - **好感度**：互动产出好感度，用于解锁角色剧情、语音、表情。
  - **专属道具**：小游戏胜利产出角色专属道具（如“安眠香薰”、“定制礼服”），可用于大厅展示或战斗增益（微量）。
- **消耗**：
  - **体力**：每次互动消耗10点体力（每日上限200点）。
  - **时间**：无冷却，但每日总互动次数上限为50次（防止刷取过度）。

### 4.3 刺激付费设计
- **预览机制**：未购买宿舍的玩家可进入“预览模式”，体验一次自由互动（限时30秒），触发角色随机语音，诱导付费。
- **好感度锁**：部分高好感度互动（如“拥抱”、“膝枕”）需好感度达到特定等级（如5000点）才可解锁，加速玩家购买宿舍后持续互动。

## 五、旧系统与数据联动 (强依赖)
### 5.1 已有系统接口
- **角色系统**：调用角色ID、模型资源、好感度数据接口。
- **体力系统**：调用体力值接口（ID: STAMINA_001）。
- **道具系统**：调用道具背包接口（ID: ITEM_BAG_001），用于存储“安眠香薰”等产出道具。

### 5.2 新增全局接口
- **互动记录表**：记录每个角色每日互动次数、好感度变化、小游戏胜利次数。
- **宿舍解锁表**：记录玩家已购买的宿舍列表。
