# 一、系统概述与设计愿景

【角色私人宿舍】系统旨在为玩家提供一个与已获得角色进行**私密、沉浸式、高互动性**的专属空间。该系统完全契合《Project T》的“ML向情感羁绊”与“极致表现层”两大支柱：
- **情感羁绊**：通过高频率、高自由度的交互（触摸、对话、小游戏），强化角色对玩家的“依赖感”与“专属感”，营造独处暧昧氛围。
- **角色表现**：所有交互均以高精度3D模型为核心，配合特写运镜与物理反馈，最大化展示角色魅力。

# 二、核心规则与玩法机制

1. **解锁条件**：玩家获得任意角色（不限稀有度）后，自动解锁该角色的私人宿舍入口。
2. **进入方式**：从主界面“基地”入口进入，选择对应角色卡片进入其专属宿舍房间。
3. **核心玩法循环**：
   - **自由探索**：玩家可在宿舍内自由拖拽视角，点击场景中的家具或角色身体部位触发交互。
   - **交互反馈**：每次交互触发对应的角色动作、语音（ASMR级别）、表情变化及物理抖动反馈。
   - **迷你游戏**：宿舍内固定放置2-3个互动小游戏（如：猜拳、拍手、喂食），购买对应道具后解锁，无养成数值，即买即玩。
   - **拍照模式**：玩家可随时进入拍照模式，自由调整镜头角度、滤镜、角色姿势（需购买姿势包），截图保存。
4. **重置规则**：无每日次数限制，所有交互与游戏可反复触发，无冷却时间。

# 三、表现层与角色展示联动 (核心强制项)

1. **UI/场景设计**：
   - 宿舍场景采用**第一人称近景视角**，角色占据屏幕中央60%以上面积，背景为温馨私密房间（床、沙发、地毯等）。
   - 交互时自动切换至**特写镜头**（如触摸脸部时镜头推近至面部，触摸腿部时镜头下移），确保玩家始终能看到角色最性感的部位。
   - 所有UI按钮（如拍照、游戏入口）采用**半透明浮动式**，绝不遮挡角色身体。

2. **擦边互动设计**：
   - **触摸反馈**：点击角色不同部位（头、肩、手、腰、腿）触发差异化反应：
     - 头部：脸红、闭眼、轻声呢喃。
     - 腰部：身体微颤、娇喘、躲避动作。
     - 腿部：害羞夹腿、裙摆物理飘动、腿部特写。
   - **物理引擎**：所有交互均触发软体物理（胸部、臀部抖动），布料（裙摆、袖口）随动作自然飘动。
   - **语音系统**：每个交互动作对应3-5条随机语音，包含ASMR级耳语、喘息、撒娇内容（如“分析员，别这样…”“好痒…”）。
   - **底线遵守**：所有互动止步于“抚摸、触碰、耳语”，**严禁**出现脱衣、露点、性行为模拟。最大尺度为“角色衣衫半褪（如肩带滑落）但关键部位完全遮挡”，或“角色躺在床上，玩家可躺在其身旁触发依偎动画”。

3. **特殊解锁内容**：
   - 购买特定家具/道具可解锁**专属互动动画**（如：一起躺在床上看星星、喂食蛋糕、梳头）。
   - 所有动画均包含**慢镜头回放**功能，方便玩家截图。

# 四、经济循环与商业化埋点

1. **付费道具分类**（均为独立直购，非抽卡）：
   - **家具包**（定价 6/12/30 美元）：包含主题家具套装（如：欧式公主床、日式榻榻米、泳池派对），每个套装附带1个专属互动动画。
   - **姿势包**（定价 3/6 美元）：解锁拍照模式中的特定姿势（如：跪坐、侧卧、回眸）。
   - **服装包**（定价 15/25 美元）：角色专属宿舍睡衣/居家服，附带全新触摸反馈语音包（比默认更亲密）。
   - **迷你游戏包**（定价 3 美元）：解锁宿舍内一个小游戏（如：猜拳、拍手游戏）。

2. **免费内容**：
   - 每个角色默认赠送基础宿舍（简单装修）与3个基础触摸互动。
   - 每日首次进入宿舍可领取少量好感度道具（用于主界面好感度系统，非本系统消耗）。

3. **付费刺激设计**：
   - 宿舍界面常驻“推荐”按钮，展示最新上架的家具/服装包，并附带角色试穿/试用动画预览。
   - 购买整套主题家具后，角色会主动发送感谢短信（模拟真实聊天），增强付费获得感。

# 五、旧系统与数据联动 (强依赖)

1. **角色数据接口**：需调用 `character_id`、`character_name`、`rarity`、`model_3d_path`、`voice_pack_id`。
2. **好感度系统接口**：需新增接口 `affection_bonus`（每日首次进入宿舍增加少量好感度）。
3. **拍照系统接口**：需新增全局截图功能，支持隐藏UI、滤镜叠加、分辨率选择。
4. **短信系统接口**：需调用 `message_system` 发送购买后感谢短信。
5. **新增全局数据表**：
   - `dorm_furniture`（家具ID、名称、价格、关联互动动画ID）
   - `dorm_pose`（姿势ID、名称、价格、模型骨骼配置）
   - `dorm_costume`（服装ID、名称、价格、模型路径、语音包ID）
   - `dorm_minigame`（小游戏ID、名称、价格、游戏逻辑脚本路径）

1. **性能风险**：宿舍场景需同时加载高精度角色模型、物理引擎、实时语音，对移动端性能压力较大。是否需要针对中低端机型做画质分级（如关闭物理、降低模型面数）？
