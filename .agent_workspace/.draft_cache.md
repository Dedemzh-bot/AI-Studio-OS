# 一、系统概述与设计愿景

【角色私人宿舍】系统旨在为玩家提供一个与已获得角色进行深度私密互动的专属空间。该系统以“触摸互动小游戏”为核心主轴，融合角色语音、动作反馈及轻量级小游戏，强化玩家与角色之间的情感羁绊（Master-Love）。所有交互设计均严格遵循“擦边媚宅”定位，通过暧昧的镜头语言、ASMR级语音和物理抖动反馈，最大化展现女性角色的魅力与陪伴感，绝不越界至R18内容。

# 二、核心规则与玩法机制

1. **解锁条件**：玩家需拥有对应角色（通过抽卡获取）后，方可解锁其私人宿舍。宿舍入口位于主界面大厅的“互动”分页。
2. **进入与退出**：点击角色头像进入其专属宿舍场景。场景为固定视角的3D房间，角色以3D模型呈现，可自由旋转视角（限制俯仰角，防止穿模）。
3. **互动方式**：
   - **触摸反馈**：玩家点击角色身体不同部位（头部、手臂、腰部、腿部等），角色会触发对应的语音、表情变化和身体动作（如害羞、躲闪、撒娇）。触摸敏感区域（如大腿、腰部）会触发更强烈的反应和特殊语音，但物理交互止步于抖动和红晕，无进一步动作。
   - **好感度等级**：互动效果受角色好感度等级影响。好感度越高，可触摸区域越多，角色反应越亲密，解锁更多特殊语音和动作。
   - **小游戏触发**：在宿舍界面中，会随机出现“互动气泡”提示。点击后触发轻量级小游戏，例如：
     - **“整理衣装”**：在限定时间内，按照提示顺序点击角色身上的装饰（如蝴蝶结、纽扣），成功则角色会做出感谢动作并赠送好感度道具。
     - **“猜猜我在哪”**：角色躲到房间某处，玩家需在30秒内通过点击场景中的可疑位置（如衣柜、窗帘后）找到她，成功则触发拥抱动画（拥抱仅限上半身，无亲吻）。
     - **“梳妆打扮”**：为角色选择不同的发饰或小饰品，角色会给出不同反应（开心、害羞、吐槽），并增加少量好感度。
4. **重置规则**：每日每个角色可进行10次基础触摸互动（每次间隔30秒），小游戏每日可触发3次（所有角色共享次数）。每日凌晨5点重置。

# 三、表现层与角色展示联动 (核心强制项)

1. **UI/场景展示**：
   - 宿舍场景采用高精度3D渲染，角色模型与战斗模型一致，但增加更多细节（如家居服、睡衣等专属服装，可通过抽卡或活动获取）。
   - 视角默认采用“男友视角”（第三人称过肩），角色面向玩家，确保玩家始终能看到角色的正面和全身。
   - 交互时，镜头会自动推近，聚焦于被触摸部位或角色面部表情，并伴有柔光特效和轻微景深效果。
2. **擦边互动与物理反馈**：
   - 触摸敏感区域时，角色身体会触发软体物理抖动（Jiggle Physics），并伴有脸红、眼神躲闪等表情变化。
   - 语音采用ASMR级别录制，包含耳语、喘息、撒娇等暧昧内容，但无任何实质性性行为描述。
   - 小游戏成功后的奖励动画（如拥抱）仅包含上半身接触，镜头从侧面或背后拍摄，避免正面特写，严格控制在“点到为止”的尺度。
3. **底线遵守**：所有互动设计均无性行为、露点或直接性暗示。触摸反馈仅停留在“害羞”和“撒娇”层面，小游戏内容健康且轻度，符合“灰色地带”定位。

# 四、经济循环与商业化埋点

1. **产出资源**：
   - 每日互动和小游戏产出少量“好感度经验”（用于提升角色好感度等级）和“宿舍币”（用于购买宿舍家具或装饰）。
   - 小游戏有概率产出“角色专属语音碎片”（集齐可解锁新语音）或“限定互动动作”。
2. **消耗资源**：
   - 宿舍币可购买家具（无属性，仅装饰）和角色专属服装（需额外付费或抽卡）。
   - 部分特殊互动区域（如“膝枕”、“摸头杀”）需要消耗“亲密券”（可通过抽卡或活动获得）。
3. **商业化刺激**：
   - 新角色上线时，同步推出其专属宿舍服装和互动动作，刺激玩家抽取角色并购买皮肤。
   - 限定小游戏（如节日主题）会产出稀有互动道具，增加玩家活跃度和付费意愿。

# 五、旧系统与数据联动 (强依赖)

1. **好感度系统**：需调用现有好感度系统接口，获取角色当前好感度等级，以决定可触摸区域和互动反馈强度。
2. **角色数据**：需读取玩家已拥有的角色列表，用于解锁宿舍入口。
3. **背包系统**：需调用背包系统，管理“宿舍币”、“亲密券”、“语音碎片”等道具的存储与使用。
4. **新增接口**：
   - `daily_interaction_count`：记录每个角色每日互动次数。
   - `mini_game_attempts`：记录每日小游戏剩余次数。
   - `unlocked_touch_zones`：记录每个角色已解锁的触摸区域。
