# 游戏设计草案：角色私人宿舍系统

## 一、系统概述与设计愿景
**系统目的**：为付费玩家提供一个高度沉浸、可互动的私人空间，让玩家与已拥有的女性角色进行亲密互动，强化“ML向的情感羁绊”核心支柱。
**契合愿景**：该系统是“极度媚宅的角色表现”的终极载体。通过第一人称视角的近距离互动、擦边小游戏和专属语音，将角色魅力从战斗场景延伸至私密空间，满足玩家对角色“占有感”和“陪伴感”的需求。系统定位为一次性付费解锁的“高级内容包”，不产生持续付费压力，符合“付费玩家”目标受众。

## 二、核心规则与玩法机制
### 2.1 参与条件
- **解锁前提**：玩家需拥有至少一名五星角色，并通过一次性付费（例如：¥68 / $9.99）购买“私人宿舍钥匙”道具，永久解锁该功能。
- **角色入住**：解锁后，玩家拥有的所有角色均可进入宿舍。每个角色拥有独立的宿舍房间，互不干扰。

### 2.2 玩法循环
1. **进入宿舍**：从主界面“基地”入口进入，选择角色后加载其专属宿舍场景。
2. **自由观赏模式**：默认状态下，角色处于待机状态（如坐在床边、站在窗前）。玩家可自由旋转/缩放视角，切换角色服装（已解锁的皮肤）、姿势（预设动作如“坐下”、“伸懒腰”）。
3. **互动模式（核心）**：玩家点击角色身体特定区域（如头部、手部、腰部），触发预设的“互动动作”与“语音反馈”。
   - **动作反馈**：角色会做出相应反应（如被摸头时脸红低头、被拍肩时转身微笑）。
   - **语音反馈**：播放专属互动语音（ASMR风格，耳语、轻笑等），每次互动有冷却时间（如5秒），防止刷屏。
4. **擦边小游戏**：每周可触发一次限时小游戏（如“猜拳”、“拍手游戏”），胜利后角色会展示特殊胜利动画（如开心拥抱、撒娇），并解锁一段专属剧情片段（Live2D演出）。
5. **好感度累积**：每次互动、小游戏胜利均增加微量“宿舍好感度”。累积到特定阈值可解锁：
   - 新互动动作（如“摸头杀”、“牵手”）。
   - 新语音台词（更亲密的称呼）。
   - 特殊待机动画（如角色主动靠近镜头）。

### 2.3 重置规则
- 互动冷却：每次互动后5秒冷却，每日总互动次数无上限。
- 小游戏：每周一重置次数（1次）。
- 好感度：不重置，永久累积。

## 三、表现层与角色展示联动 (核心强制项)
### 3.1 场景与UI设计
- **第一人称视角**：默认采用“玩家主观视角”，模拟玩家站在角色面前。屏幕中央无任何UI遮挡，角色占据画面70%以上。
- **运镜设计**：进入宿舍时，自动播放一段“开门动画”，镜头从角色脚部缓慢上移至面部，强调身材曲线。互动时，镜头自动拉近至角色上半身特写。
- **物理反馈**：角色模型启用高规格软体物理（胸部、臀部、头发），在互动动作中产生自然抖动。服装材质支持透视/湿润效果（如汗水、水渍）。

### 3.2 擦边互动设计（严格遵守红线）
- **互动区域限制**：可点击区域仅限于头部、肩膀、手臂、腰部、手部。**严禁**点击胸部、臀部、大腿内侧等敏感区域。若玩家尝试点击禁区，角色会后退一步并摇头，播放“不可以哦~”的语音，无惩罚但无奖励。
- **动作尺度**：互动动作止步于“拥抱”、“摸头”、“牵手”、“击掌”。**严禁**出现亲吻、抚摸敏感部位、脱衣等R18动作。
- **小游戏尺度**：游戏内容为纯情向（如猜拳、拍手），胜利动画为角色开心跳跃或轻轻拥抱玩家（镜头从背后拍摄拥抱场景，不展示正面接触）。**严禁**出现任何性暗示动作。
- **语音尺度**：语音内容为“你真好~”、“最喜欢你了”、“今天想我了吗？”等暧昧台词，语气轻柔带喘息感。**严禁**出现直接性邀请或脏话。

## 四、经济循环与商业化埋点
### 4.1 付费点
- **一次性解锁**：“私人宿舍钥匙”道具，售价¥68，永久解锁所有角色宿舍功能。
- **皮肤联动**：宿舍中可展示玩家已购买的角色皮肤，新皮肤上架时，宿舍场景会同步更新对应互动动作（如泳装皮肤解锁“泼水”互动），刺激皮肤购买。

### 4.2 资源产出与消耗
- **产出**：宿舍好感度（仅用于解锁宿舍内容，不与其他系统共享）。
- **消耗**：无。系统设计为纯内容消费，不消耗游戏内货币或体力。

### 4.3 刺激抽卡/皮肤购买
- 新角色上线时，其宿舍互动内容作为“角色魅力展示”的一部分，在抽卡预览中播放片段，吸引玩家抽取。
- 限定皮肤附带“宿舍专属互动动作”，仅在宿舍中可触发，强化皮肤价值。

## 五、旧系统与数据联动 (强依赖)
### 5.1 需调用的已有系统
- **角色数据系统**：读取玩家拥有的角色列表、星级、已解锁皮肤。
- **好感度系统**：需新增接口 `get_dormitory_affection(role_id)` 和 `add_dormitory_affection(role_id, value)`，与主好感度系统独立。
- **语音系统**：需新增宿舍专属语音资源池，按好感度等级解锁。

### 5.2 需新增的全局接口
- `is_dormitory_unlocked(player_id)`：判断玩家是否购买钥匙。
- `get_dormitory_interaction_list(role_id, affection_level)`：返回当前可用的互动动作与语音列表。
