# 游戏设计文档草案：角色私人宿舍系统

## 一、系统概述与设计愿景

本系统旨在为玩家提供一个与已获取角色进行深度亲密互动的私密空间。核心定位为“触摸互动小游戏”，强调ML向的情感羁绊与擦边表现。玩家可通过点击、滑动等操作与角色进行亲密互动，触发丰富的语音、表情和物理反馈，并解锁特殊的小游戏环节。该系统完全契合《项目宪法》中“ML向的情感羁绊”与“极致的表现层”两大支柱，将角色魅力从战斗场景延伸至日常陪伴，强化玩家的拥有感与代入感。

## 二、核心规则与玩法机制

1.  **解锁条件**：玩家需拥有对应角色（通过抽卡获取）后，方可解锁该角色的私人宿舍。宿舍为独立场景，每个角色拥有专属房间。
2.  **进入方式**：在大厅或角色详情界面，提供“前往宿舍”入口。
3.  **互动模式**：
    *   **自由触摸模式**：玩家可自由点击/滑动角色身体的不同部位（如头部、脸颊、手、肩膀等）。每次触摸都会触发对应的语音反馈（如娇嗔、轻笑、害羞）、表情变化（脸红、眨眼）以及物理抖动（如胸部、头发的软体物理）。
    *   **互动道具模式**：玩家可使用通过游戏内活动或付费获得的“互动道具”（如羽毛、梳子、小零食），对角色进行特定互动。道具会触发更强烈的反馈和专属语音。
4.  **小游戏环节**：
    *   **触发条件**：当玩家与角色的亲密度达到一定等级，或使用特定道具后，可触发小游戏。
    *   **游戏类型**：例如“默契问答”（角色提问，玩家选择答案）、“心跳按摩”（在限定时间内，按照提示顺序点击角色特定部位）等。小游戏胜利后，角色会展示特殊胜利动画并解锁新的互动语音/动作。
5.  **亲密度系统**：每次互动（触摸、使用道具、完成小游戏）都会增加该角色的亲密度。亲密度等级提升会解锁更多互动区域、更亲密的语音、更丰富的表情以及新的小游戏。
6.  **重置规则**：每日互动次数无硬性限制，但每日可获得的亲密度上限受玩家等级或VIP等级影响。小游戏每日有固定挑战次数。

## 三、表现层与角色展示联动 (核心强制项)

1.  **场景与视角**：
    *   宿舍场景采用第一人称视角，玩家仿佛就坐在角色对面。角色以3D模型呈现在屏幕中央，占据画面70%以上。
    *   场景布置温馨私密（如床、沙发、落地灯），但所有家具均为背景，不提供交互，确保玩家注意力100%集中在角色身上。
2.  **触摸反馈**：
    *   **视觉**：触摸部位会高亮，并伴有微弱的泛红效果。角色会做出相应的肢体动作（如被摸头时微微低头，被摸手时轻轻缩回）。
    *   **听觉**：每个触摸点对应至少3条不同语音，随机播放，避免重复感。语音风格为ASMR级别，轻声细语，充满暧昧感。
    *   **物理**：触摸胸部、臀部等敏感区域时，会触发高规格的软体碰撞物理（Jiggle Physics），并伴有角色害羞或嗔怪的反应，严格控制在“擦边”尺度内，无任何实质性行为表现。
3.  **特殊运镜**：
    *   在小游戏环节或使用特定道具时，镜头会自动拉近，特写角色的面部表情（如害羞、惊喜）或身体局部（如手指、嘴唇），增强沉浸感与暧昧氛围。
4.  **底线遵守**：所有互动设计均止步于“性暗示”与“暧昧氛围”，无任何露点、性行为或直接描写。物理抖动和语音反馈均以“点到为止”为原则，给玩家留足遐想空间。

## 四、经济循环与商业化埋点

1.  **核心付费点**：
    *   **互动道具**：通过商城出售“互动道具礼包”（如“甜蜜梳妆盒”、“调皮羽毛套装”），内含多种限定互动道具。道具为消耗品，使用后增加大量亲密度并触发特殊反馈。
    *   **宿舍皮肤**：出售不同主题的宿舍背景皮肤（如“和风温泉”、“欧式公主房”），皮肤本身不提供数值，但会改变场景氛围，并解锁该场景下的专属互动动作与语音。
2.  **产出资源**：
    *   每日首次进入宿舍、完成小游戏可获得少量“亲密度礼物”（用于提升任意角色亲密度）和“互动币”（用于在宿舍专属商店兑换基础互动道具）。
3.  **商业化刺激**：新角色上线时，同步推出该角色的专属互动道具和宿舍皮肤，刺激玩家抽取角色后进行付费体验。

## 五、旧系统与数据联动 (强依赖)

1.  **角色系统**：需要调用角色ID、角色模型资源、角色语音资源。
2.  **好感度系统**：需要新增“亲密度”接口（ID: INTIMACY_LEVEL），与现有好感度系统并行或作为其子模块。亲密度等级提升可解锁新的好感度剧情或语音。
3.  **背包系统**：需要新增“互动道具”和“互动币”的ID与存储接口。
4.  **商城系统**：需要新增“互动道具礼包”和“宿舍皮肤”的商品ID与购买接口。
