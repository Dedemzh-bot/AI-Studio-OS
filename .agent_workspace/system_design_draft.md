# 公会悬赏任务系统 - 系统设计草案

## 1. 系统概述

- **系统名称**：公会悬赏任务系统
- **系统定位**：公会核心日常玩法，为公会成员提供每日可重复完成的任务，驱动公会资金增长与个人收益
- **核心玩法闭环**：每日系统自动刷新一批星级悬赏任务 → 玩家接取任务 → 完成任务获得个人金币与公会贡献度 → 公会资金累积用于科技升级与商店兑换 → 玩家可消耗钻石手动刷新任务列表以获取更高星级任务

## 2. 模块划分

- **任务池管理模块**：负责每日悬赏任务的生成、刷新与星级分配。每日凌晨5点自动从任务模板库中抽取一批任务，按概率分布分配1至5星。与任务模板库模块联动获取任务类型与基础数据
- **任务模板库模块**：存储所有悬赏任务类型模板，包括打怪、收集、捐献等类型，定义每个模板的基础参数如目标数量、基础奖励系数。为任务池管理模块提供数据源
- **玩家任务接取模块**：管理玩家接取任务的状态，包括已接取、进行中、已完成、已领取奖励。限制玩家同时接取的任务数量上限。与任务池管理模块交互获取可接取任务列表
- **任务完成与奖励发放模块**：验证任务完成条件，发放个人奖励（金币、公会贡献度）与公会资金。与公会货币系统联动，调用公会贡献度与公会钻石的数值接口
- **手动刷新模块**：允许玩家每日消耗钻石手动刷新任务列表，刷新后重新生成一批任务。与任务池管理模块联动，消耗钻石后触发重新抽取逻辑，重新抽取要排除到当前已经抽出的任务
- **公会资金管理模块**：记录公会资金总量，资金来源于任务完成后的公会资金增加。与公会科技节点模块和公会商店模块联动，用于科技升级与商店兑换消耗

## 3. 数值成长维度

- **任务星级**：离散解锁维度，取值范围1至5星，每个星级对应不同的奖励系数与刷新概率
- **每日任务刷新次数上限**：连续成长维度，随玩家等级递增，取值范围建议为3至10次
- **同时接取任务数量上限**：连续成长维度，随玩家等级递增，取值范围建议为2至6个
- **手动刷新钻石消耗**：离散解锁维度，根据刷新次数递增消耗，首次消耗50钻石，后续每次增加20钻石，上限为200钻石
- **任务基础奖励系数**：离散解锁维度，按星级设定，1星系数1.0，2星系数1.5，3星系数2.5，4星系数4.0，5星系数6.0
- **公会资金增加比例**：离散解锁维度，按星级设定，1星增加10资金，2星增加25资金，3星增加50资金，4星增加100资金，5星增加200资金

## 4. 核心字段说明

- **任务池管理模块**
  - `task_id`：整型，任务实例唯一标识，示例值10001
  - `template_id`：整型，关联任务模板库的模板ID，示例值201
  - `star_level`：整型，任务星级，取值范围1至5，示例值3
  - `refresh_date`：字符串，任务生成日期，格式YYYY-MM-DD，示例值2025-04-10
  - `is_active`：布尔型，任务是否处于可接取状态，示例值true

- **任务模板库模块**
  - `template_id`：整型，模板唯一标识，示例值201
  - `task_type`：字符串，任务类型，可选值kill、collect、donate，示例值kill
  - `target_count`：整型，任务目标数量，示例值10
  - `base_reward_gold`：整型，基础金币奖励，示例值500
  - `base_reward_contribution`：整型，基础公会贡献度奖励，示例值50
  - `base_guild_fund`：整型，基础公会资金增加量，示例值10

- **玩家任务接取模块**
  - `player_id`：整型，玩家唯一标识，示例值10001
  - `task_id`：整型，已接取的任务实例ID，示例值10001
  - `accept_time`：字符串，接取时间，格式YYYY-MM-DD HH:MM:SS，示例值2025-04-10 10:30:00
  - `progress`：整型，当前完成进度，示例值5
  - `status`：字符串，任务状态，可选值accepted、in_progress、completed、rewarded，示例值in_progress

- **手动刷新模块**
  - `player_id`：整型，玩家唯一标识，示例值10001
  - `refresh_date`：字符串，刷新日期，格式YYYY-MM-DD，示例值2025-04-10
  - `refresh_count`：整型，当日已刷新次数，示例值2
  - `diamond_cost`：整型，本次刷新消耗钻石数，示例值70

- **公会资金管理模块**
  - `guild_id`：整型，公会唯一标识，示例值5001
  - `total_fund`：整型，公会当前资金总量，示例值50000
  - `daily_income`：整型，当日累计增加资金，示例值1500

## 5. 系统流程

- **每日任务刷新流程**：每日凌晨5点，系统自动触发任务池管理模块，从任务模板库中随机抽取10个任务模板，按概率分布分配星级，1星概率40%，2星概率30%，3星概率20%，4星概率8%，5星概率2%。生成任务实例并标记为活跃状态，存入任务池。所有公会成员均可查看当前活跃任务列表

- **玩家接取任务流程**：玩家进入公会大厅，查看当前活跃任务列表。点击接取按钮，系统检查玩家当前已接取任务数量是否达到上限，若未达到则创建接取记录，状态设为accepted，任务从活跃列表移除。若达到上限则提示玩家先完成或放弃已有任务

- **任务完成与奖励发放流程**：玩家完成任务目标后，系统自动检测进度达到target_count，将任务状态更新为completed。玩家点击领取奖励按钮，系统发放个人金币与公会贡献度，同时公会资金增加对应数值。任务状态更新为rewarded，玩家接取记录关闭

- **手动刷新任务列表流程**：玩家在任务列表界面点击刷新按钮，系统检查当日刷新次数是否达到上限。若未达到，计算本次消耗钻石数，扣除玩家钻石后触发任务池重新抽取逻辑，生成新的任务列表替换当前活跃任务。刷新次数加一，消耗钻石数按递增规则更新。若达到上限则提示玩家次日再试
