# 账号信息系统 - 宏观设计草案

## 一、系统概述与设计愿景
本系统定位为纯展示型玩家信息面板，核心目的是为玩家提供一个清晰、美观的个人身份标识界面。系统仅展示玩家基础信息（如昵称、头像、等级、UID、注册天数等），不承载任何社交互动、好友查看、名片分享、签名/称号自定义或身份成长绑定功能。设计上强调信息展示的简洁性与视觉愉悦感，同时严格遵守“不遮挡角色模型”的UI红线。

## 二、核心规则与玩法机制
- **信息展示维度**：固定展示以下字段：玩家昵称、头像（静态/动态）、当前等级、UID、注册天数、最后登录时间。
- **无交互操作**：系统为纯只读界面，不提供任何编辑、分享、点赞、留言等交互入口。
- **入口位置**：主界面左上角或右上角，点击头像或等级图标进入。
- **无成长/成就绑定**：不关联任何成就系统、收藏品、称号或准入凭证。

## 三、表现层与角色展示联动 (核心强制项)
- **UI布局**：信息面板采用半透明毛玻璃背景，置于屏幕一侧（如左侧），主体区域保留给当前展示角色（大厅模型/立绘）。面板尺寸不超过屏幕宽度的1/3，高度不超过屏幕高度的1/2，确保不遮挡角色关键部位（面部、胸部、腿部）。
- **角色展示**：进入账号信息界面时，自动切换至当前编队主控角色的特写镜头（半身或全身），镜头角度可轻微旋转（15度范围），突出角色魅力。面板信息浮于角色一侧，不遮挡角色主体。
- **动态头像**：头像支持使用角色动态立绘或Live2D，点击头像可触发角色专属语音（如“分析员，你在看我的资料吗？”），增强陪伴感。
- **物理反馈**：进入界面时，角色模型触发一次轻微呼吸或头发飘动动画，无额外擦边互动。

## 四、经济循环与商业化埋点
- **无直接付费点**：本系统为纯展示，不产出任何资源，不消耗任何资源。
- **间接商业化**：动态头像框、角色专属头像框可作为付费皮肤（抽卡/直购）的附属品，在账号信息界面展示，刺激玩家收集欲。

## 五、旧系统与数据联动 (强依赖)
- **玩家基础数据**：从账号数据库读取昵称、UID、注册时间、最后登录时间。
- **等级系统**：读取玩家当前等级（来自经验系统）。
- **头像系统**：读取玩家当前选中的头像（来自头像/头像框系统）。
- **角色模型**：读取当前编队主控角色ID，调用大厅角色展示系统。

## 六、待确认风险与追问
- 无（方向已由老板明确为纯展示型，无需进一步追问）。
