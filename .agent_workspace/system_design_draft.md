# 一、系统概述与设计愿景

【角色私人宿舍】系统是一个**买断制解锁**的、为每位女性角色量身定制的**专属私密空间**。其核心设计愿景是：将角色从“战斗工具”升维为“可陪伴的恋人”。通过高度定制化的房间装修、专属好感度对话链、特殊触摸反馈及私密事件，深度强化玩家与单一角色的情感羁绊（Master-Love），完美契合《Project T》中“极致表现层”与“ML向情感羁绊”的核心支柱。该系统是游戏内除战斗外，最重要的角色魅力展示与情感付费转化场景。

# 二、核心规则与玩法机制

1.  **解锁条件**：
    *   玩家需在商城中**一次性付费购买**“私人宿舍”核心模块（定价策略见第五节）。
    *   购买后，玩家拥有的**每一位**女性角色将自动解锁其对应的**专属私人房间**。
    *   若后续获得新角色，其房间将自动解锁，无需再次付费。

2.  **玩法循环**：
    *   **进入**：从主界面“宿舍”入口进入，选择角色，进入其专属房间。
    *   **核心互动**：
        *   **自由视角**：玩家可自由旋转、缩放视角，从任意角度欣赏角色。
        *   **触摸反馈**：点击角色身体不同部位（头、手、脸等），触发不同等级的语音、表情和动作反馈（严格遵守擦边底线）。
        *   **好感度对话**：随着角色好感度等级提升，解锁新的专属对话选项，触发更亲密的剧情片段。
        *   **换装展示**：角色会穿着玩家当前为其装备的服装（包括皮肤），在房间内进行待机动作展示。
    *   **长期目标**：通过日常互动提升角色好感度，解锁更多房间家具、互动姿势和私密剧情。

3.  **重置与状态**：
    *   好感度互动每日有次数上限，次日重置。
    *   房间状态（家具摆放、灯光等）由玩家自定义，永久保存。

# 三、表现层与角色展示联动 (核心强制项)

1.  **最大化角色展示**：
    *   **专属运镜**：进入房间时，自动播放一段特写镜头，从角色脚部缓慢上移至面部，随后切换至玩家可控视角。
    *   **UI 极简化**：互动时，所有 UI 按钮半透明化或隐藏，确保屏幕 90% 以上区域用于展示角色模型。
    *   **动态环境**：房间内光源（如窗外的月光、台灯）会随角色动作产生动态光影，凸显角色轮廓与材质细节。

2.  **擦边互动与物理反馈**：
    *   **触摸互动**：触摸角色不同部位，触发对应的 Jiggle Physics（如胸部、臀部抖动）、脸红表情和 ASMR 级别耳语语音。
    *   **特殊姿势**：好感度达到特定等级后，解锁“坐在床边”、“慵懒躺卧”等更具亲密感的待机姿势。
    *   **服装反馈**：穿着特定皮肤时，触摸反馈和语音会有专属变化（例如：泳装皮肤的触摸反应更敏感）。

3.  **底线遵守**：
    *   所有互动严格控制在“性暗示”与“暧昧氛围”层面。触摸反馈仅限语音、表情和物理抖动，**绝不涉及任何脱衣、露点或模拟性行为**的动画或逻辑。所有剧情对话止步于“恋人未满”或“极度暧昧”的尺度。

# 四、经济循环与商业化埋点

1.  **付费点**：
    *   **核心买断**：一次性付费解锁“私人宿舍”系统本体。
    *   **皮肤驱动**：宿舍系统是展示新皮肤的最佳场景，高品质皮肤（含专属宿舍互动姿势、语音）将成为核心付费动力。
    *   **家具礼包**（可选）：出售特定主题的家具套装（如“和风”、“现代简约”），仅改变房间装饰，不影响数值，纯外观付费。

2.  **资源产出**：
    *   **好感度经验**：每日互动产出少量好感度经验，用于解锁剧情和互动。
    *   **无抽卡资源产出**：该系统不产出任何抽卡资源，以维持其“纯情感陪伴”的定位，避免与核心抽卡系统冲突。

# 五、旧系统与数据联动 (强依赖)

1.  **角色系统**：需调用角色 ID、角色模型、角色当前皮肤 ID。
2.  **好感度系统**：需调用或新增全局好感度等级接口，用于解锁宿舍内的专属对话和互动姿势。
3.  **商城系统**：需新增“私人宿舍”买断制商品 ID 及“家具礼包”商品 ID。
4.  **新增全局接口**：
    *   `PlayerDormitoryData`：存储玩家已解锁的宿舍功能、每个角色的好感度进度、房间自定义数据（家具摆放、灯光设置等）。
