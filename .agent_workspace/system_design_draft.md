# 公会后花园钓鱼系统 - 系统设计草案

## 1. 系统概述

- **系统名称**：公会后花园钓鱼系统
- **系统定位**：公会休闲玩法，消耗公会贡献度换取公会资金，提供轻度随机产出体验
- **核心玩法闭环**：玩家消耗公会贡献度购买特制鱼饵 → 在已解锁的水域中钓鱼 → 获得不同品质的鱼 → 上交鱼获转化为公会资金 → 公会资金用于公会发展，形成资源循环

## 2. 模块划分

### 2.1 水域管理模块
- **职责**：管理三个水域的解锁条件、开放状态、鱼获产出配置
- **与其他模块关系**：依赖公会等级系统判断解锁状态，为钓鱼流程模块提供当前可钓鱼的水域列表

### 2.2 鱼饵商店模块
- **职责**：提供特制鱼饵的购买功能，消耗公会贡献度（currency_id: 1）
- **与其他模块关系**：调用公会货币系统扣除贡献度，为钓鱼流程模块提供鱼饵库存校验

### 2.3 钓鱼流程模块
- **职责**：处理钓鱼操作的核心逻辑，包括鱼饵消耗、随机鱼获产出、品质判定
- **与其他模块关系**：从水域管理模块获取当前水域的产出配置，从鱼饵商店模块校验鱼饵数量，产出结果传递给鱼获仓库模块

### 2.4 鱼获仓库模块
- **职责**：存储玩家钓到的鱼，按品质分类管理，提供上交功能
- **与其他模块关系**：接收钓鱼流程模块的产出结果，上交时调用公会资金系统增加公会资金

### 2.5 公会资金转化模块
- **职责**：处理鱼获上交到公会资金的转化逻辑，根据鱼品质计算转化比例
- **与其他模块关系**：从鱼获仓库模块获取待上交的鱼，调用公会资金系统增加资金，同时从鱼获仓库中扣除对应鱼获

## 3. 数值成长维度

### 3.1 连续成长维度
- **水域解锁等级**：公会等级达到特定值后解锁对应水域，解锁后永久可用
  - 取值范围：浅水区（公会等级 1 解锁）、深水区（公会等级 3 解锁）、神秘区（公会等级 5 解锁）
- **鱼饵价格**：随公会等级提升可能微调，建议保持固定或小幅递增
  - 取值范围：10 ~ 50 公会贡献度/个

### 3.2 离散解锁维度
- **水域解锁**：公会等级达到 1、3、5 时分别解锁浅水区、深水区、神秘区
- **鱼品质解锁**：浅水区仅产出普通鱼，深水区产出普通+稀有鱼，神秘区产出全部三种品质
- **鱼饵购买权限**：公会等级达到 1 时解锁购买权限

## 5. 系统流程

### 5.1 鱼饵购买流程
1. 玩家在公会界面进入钓鱼系统，点击鱼饵商店
2. 系统校验玩家所在公会等级是否达到 1 级（解锁购买权限）
3. 玩家选择购买数量，系统校验玩家当前公会贡献度是否足够
4. 系统调用公会货币系统，扣除对应数量的公会贡献度（currency_id: 1）
5. 系统在玩家鱼饵持有表中增加对应数量的特制鱼饵
6. 前端刷新显示玩家当前鱼饵数量和剩余贡献度

### 5.2 钓鱼操作流程
1. 玩家在钓鱼界面选择已解锁的水域（系统根据公会等级展示可用的水域列表）
2. 系统校验玩家鱼饵持有量是否大于 0
3. 系统扣除 1 个特制鱼饵
4. 系统根据所选水域的 fish_quality_weights 配置，通过随机算法判定本次产出鱼的品质
5. 系统根据品质和当前水域的 fish_config 配置，随机选择该品质下的一种鱼
6. 系统在玩家鱼获仓库表中增加对应鱼获数量 1
7. 前端播放钓鱼动画并展示获得的鱼获信息

### 5.3 鱼获上交流程
1. 玩家在鱼获仓库界面选择要上交的鱼（支持单条或批量选择）
2. 系统校验玩家鱼获仓库中对应鱼获数量是否充足
3. 系统根据鱼获配置表中的 convert_guild_fund 值，计算总转化公会资金量
4. 系统从玩家鱼获仓库中扣除对应数量的鱼获
5. 系统调用公会资金系统，增加对应数量的公会资金
6. 前端刷新显示玩家当前鱼获库存和公会资金变化
