# 角色私人宿舍 - 宏观设计草案

## 一、系统概述与设计愿景
**核心设计目的：** 打造一个深度沉浸、高度私密的“角色专属空间”，让玩家与心爱的美少女角色在战斗之外建立强烈的情感羁绊。本系统是“极度媚宅”愿景的终极落地场景，强调陪伴感、占有感与视觉享受。
**系统定位：** 纯内容买断制（DLC式）的独立情感养成模块。作为游戏核心付费DLC之一，一次性付费解锁全部内容，无后续内购。系统不与游戏内抽卡/养成资源挂钩，专注于提供纯粹的角色互动体验。

## 二、核心规则与玩法机制
### 宏观玩法循环
1.  **进入与选择：** 玩家从主界面进入“私人宿舍”入口，选择一位已拥有的角色进入其专属房间。
2.  **自由探索与互动：** 在3D房间内，玩家可自由旋转/缩放视角，点击场景中的热点（如沙发、床、书桌、窗户）触发角色特定互动动画与对话。
3.  **小游戏挑战：** 每个角色拥有2-3个专属小游戏（如换装搭配、料理制作、射击打靶、节奏音游等），完成小游戏可解锁角色专属剧情片段、特殊互动姿势或纪念品。
4.  **情感积累：** 通过日常互动与小游戏，积累角色“好感度”。好感度解锁更亲密的对话、互动动作及房间装饰变化。
5.  **离线陪伴：** 角色在宿舍内拥有AI驱动的日常行为（看书、发呆、做家务），玩家可随时观看，增强陪伴感。

### 子系统/功能模块清单
- **角色房间系统：** 每个角色拥有独立、风格化、可轻度自定义（如更换家具颜色、摆件）的3D房间。
- **互动热点系统：** 房间内预设多个可交互点，触发角色特定动画与语音。
- **小游戏系统：** 每个角色专属的2-3个轻量级小游戏，玩法与角色人设强相关。
- **好感度系统：** 线性成长，通过互动与小游戏积累，解锁内容。
- **剧情解锁系统：** 好感度或小游戏成就解锁的短篇剧情片段（含Live2D或3D演出）。
- **纪念品系统：** 完成特定挑战获得的装饰性物品，可摆放在房间内。
- **AI日常行为系统：** 角色在房间内自动进行预设行为，营造生活感。
- **拍照模式：** 玩家可自由调整视角、滤镜、角色姿势与表情，截图保存。

## 三、表现层与角色展示联动 (核心强制项)
- **极致角色展示：** 房间内角色模型精度高于战斗模型，支持更细腻的面部表情、皮肤材质（如汗水、红晕）与服装物理（如裙摆、发丝）。
- **特殊运镜：** 互动动画强制采用“第一人称”或“近景特写”视角，聚焦角色面部、身体曲线与服装细节。例如：坐在沙发时，镜头从玩家视角仰视角色；躺在床上时，镜头俯视角色。
- **擦边互动暗示：** 互动动作设计在“灰色地带”内，如：轻抚头发、整理衣领、喂食、依靠肩膀等。严禁直接性行为或露点。
- **物理反馈：** 角色模型搭载高规格软体物理（Jiggle Physics），在互动、转身、跳跃时产生自然抖动。
- **UI最小化：** 互动时UI完全隐藏，仅保留极简的退出按钮。好感度、任务等UI以“手机消息”或“角色对话气泡”形式自然融入场景。
- **声音陪伴：** 角色在房间内会随机触发自言自语、哼歌、呼唤玩家昵称等语音，增强沉浸感。

## 四、经济循环与商业化埋点
- **付费模式：** 纯内容买断制。玩家一次性支付固定金额（如$9.99-$14.99）解锁整个“角色私人宿舍”DLC，包含所有已拥有及未来新增角色的基础互动与小游戏。
- **经济循环：** 系统内无任何游戏内货币（金币、钻石）产出或消耗。所有解锁（剧情、纪念品）仅依赖“好感度”与“小游戏成就”驱动。
- **商业化埋点：** 无内购。但可作为“豪华版/典藏版”游戏捆绑包的一部分销售，或作为新玩家入坑的吸引力点（“买断DLC，永久畅玩”）。

## 五、旧系统与数据联动 (强依赖)
- **角色解锁联动：** 玩家必须在主游戏中通过抽卡或活动获得该角色，才能在“私人宿舍”中解锁其房间。**（强依赖）**
- **角色外观联动：** 玩家在宿舍中可切换该角色已拥有的所有皮肤（包括付费皮肤），皮肤在宿舍内拥有专属互动动画与语音。**（强依赖）**
- **好感度联动：** 宿舍内的好感度独立于主游戏的好感度系统，但宿舍好感度达到特定等级，可在主游戏内解锁一个专属头像框或名片背景。**（弱联动，正向激励）**
- **剧情联动：** 宿舍内解锁的剧情片段，可在主游戏的“档案”系统中回顾。**（弱联动）**

## 六、待确认风险与追问
- **内容量风险：** 纯买断制下，每个角色需制作大量高质量互动内容（动画、语音、小游戏），开发成本极高。需确认初始版本包含的角色数量（如：首发5-8个角色）及后续更新计划（是否免费更新新角色内容？）。
- **付费意愿风险：** 玩家是否愿意为“纯情感体验”内容额外付费？需通过市场调研或A/B测试验证定价策略。
- **技术性能风险：** 高精度模型、物理模拟与AI日常行为对移动端性能压力较大。需确认目标平台的最低配置要求。
