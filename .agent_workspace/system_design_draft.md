# 角色私人宿舍系统设计文档

## 1. 系统概述

- **系统名称**：角色私人宿舍（Character Private Dormitory）
- **系统定位**：买断制角色互动系统，玩家通过购买解锁特定角色的私人宿舍空间，在宿舍内与角色进行触摸、滑动、拖拽等交互操作，触发角色动作、语音反馈及小游戏。
- **核心玩法闭环**：玩家购买宿舍 → 进入宿舍空间 → 通过点击/滑动/拖拽与角色互动 → 触发角色动作/语音/小游戏 → 获得好感度/成就/特殊奖励 → 解锁更多互动内容 → 继续探索新互动。

## 2. 详细玩法规则

### 2.1 宿舍购买与解锁
- **触发条件**：玩家等级达到 10 级，且拥有对应角色（角色已获取）。
- **判定逻辑**：
  - 每个角色独立计算宿舍解锁状态。
  - 玩家消耗指定货币（如钻石或专属宿舍券）购买宿舍。
  - 购买后永久解锁该角色宿舍。
- **成功条件**：货币足够且角色已拥有 → 购买成功，宿舍解锁。
- **失败条件**：货币不足或角色未拥有 → 购买失败，提示具体原因。

### 2.2 进入宿舍
- **触发条件**：玩家在角色详情页或宿舍列表中选择已解锁的宿舍。
- **判定逻辑**：
  - 检查该角色宿舍是否已解锁。
  - 检查当前是否有其他角色宿舍正在加载（防止多开）。
- **成功条件**：宿舍已解锁且无冲突 → 加载宿舍场景，角色出现在预设位置。
- **失败条件**：宿舍未解锁 → 跳转至购买界面。

### 2.3 基础交互操作
#### 2.3.1 点击交互
- **触发条件**：玩家在宿舍场景中点击角色身体任意部位。
- **判定逻辑**：
  - 检测点击位置是否落在角色碰撞区域内。
  - 根据点击部位（头部、手臂、腹部、腿部等）触发不同动作和语音。
  - 每个部位有冷却时间（5秒），防止连续触发。
- **成功条件**：点击位置有效且冷却结束 → 播放对应动作动画 + 语音台词。
- **失败条件**：冷却中 → 无反馈或播放轻推动作。

#### 2.3.2 滑动交互
- **触发条件**：玩家在角色身体上按住并滑动。
- **判定逻辑**：
  - 检测滑动起点和终点是否都在角色碰撞区域内。
  - 滑动距离超过阈值（50像素）视为有效滑动。
  - 根据滑动方向（上/下/左/右）触发不同反应。
- **成功条件**：滑动有效 → 角色做出对应动作（如被挠痒、被抚摸等）+ 语音反馈。
- **失败条件**：滑动距离不足或起点不在角色上 → 无反馈。

#### 2.3.3 拖拽交互
- **触发条件**：玩家长按角色（超过1秒）并拖动。
- **判定逻辑**：
  - 长按1秒后进入拖拽模式，角色跟随手指移动。
  - 拖拽到特定家具上方（如床、沙发、椅子）时，检测家具交互区域。
  - 松开手指时，判断角色是否落在家具交互区域内。
- **成功条件**：角色被拖拽到家具交互区域并松开 → 触发家具专属互动动作（如丢到床上、按到沙发上）。
- **失败条件**：松开时不在任何家具交互区域 → 角色回到原位，播放轻微抗议动作。

### 2.4 小游戏触发与规则
#### 2.4.1 挠痒小游戏
- **触发条件**：在角色腹部区域快速连续点击3次（每次间隔不超过0.5秒）。
- **判定逻辑**：
  - 记录点击时间戳，检查连续3次点击是否在1.5秒内完成。
  - 所有点击必须落在腹部碰撞区域内。
- **游戏规则**：
  - 小游戏开始后，角色会随机移动躲避。
  - 玩家需要在10秒内点击角色身体任意部位累计10次。
  - 每次点击成功增加1分，点击失败（点击空白处）不扣分。
- **成功条件**：10秒内累计点击10次 → 小游戏胜利，角色求饶动画 + 特殊语音 + 好感度+5。
- **失败条件**：10秒内未达到10次 → 小游戏失败，角色得意动画 + 普通语音。

#### 2.4.2 捉迷藏小游戏
- **触发条件**：在宿舍中连续点击3个不同家具（每个家具间隔不超过2秒）。
- **判定逻辑**：
  - 记录点击的家具ID，检查是否连续点击了3个不同家具。
  - 3次点击总时间不超过6秒。
- **游戏规则**：
  - 角色躲到随机一个家具后面（共5个可躲藏位置）。
  - 玩家需要在15秒内点击正确的家具找到角色。
  - 每次点错会扣除2秒剩余时间。
- **成功条件**：在时间内点击正确家具 → 找到角色，播放惊喜动画 + 好感度+3。
- **失败条件**：时间耗尽或点错3次 → 角色自己出来，播放无聊动画。

### 2.5 好感度系统
- **触发条件**：完成任意交互操作或小游戏。
- **判定逻辑**：
  - 每次成功交互（点击/滑动/拖拽）增加好感度+1。
  - 小游戏胜利增加额外好感度（挠痒+5，捉迷藏+3）。
  - 好感度每日上限为30点。
- **成功条件**：交互成功 → 好感度增加。
- **失败条件**：达到每日上限 → 提示“今日互动已满，明天再来吧”。

### 2.6 成就系统
- **触发条件**：累计完成特定交互次数或达成特殊条件。
- **判定逻辑**：
  - 记录每种交互的累计次数。
  - 记录小游戏胜利次数。
  - 记录拖拽到特定家具的次数。
- **成就列表**：
  - 点击达人：累计点击100次 → 解锁特殊点击动作。
  - 滑动高手：累计滑动50次 → 解锁特殊滑动语音。
  - 拖拽狂魔：累计拖拽30次 → 解锁特殊拖拽动画。
  - 小游戏冠军：累计小游戏胜利20次 → 解锁隐藏小游戏。
  - 家具大师：将角色拖拽到所有家具上各5次 → 解锁家具互动合集。

## 3. 模块划分与条件分支

### 3.1 宿舍购买模块
- **职责**：处理宿舍购买逻辑，验证玩家资格。
- **与其他模块关系**：购买成功后通知宿舍加载模块解锁状态变更。
- **条件分支**：
  - if 玩家等级 >= 10 → 允许进入购买界面
  - if 玩家等级 < 10 → 提示“需要达到10级才能购买宿舍”
  - if 角色已拥有 → 显示购买按钮
  - if 角色未拥有 → 显示“请先获取该角色”
  - if 货币足够 → 购买成功，扣除货币
  - if 货币不足 → 提示“货币不足，请充值”

### 3.2 宿舍加载模块
- **职责**：加载宿舍场景、角色模型、家具布局。
- **与其他模块关系**：从购买模块获取解锁状态，加载完成后通知交互模块。
- **条件分支**：
  - if 宿舍已解锁 → 加载完整场景
  - if 宿舍未解锁 → 跳转至购买界面
  - if 当前有其他宿舍正在加载 → 提示“请先退出当前宿舍”
  - if 加载超时（>5秒） → 显示加载失败提示，提供重试按钮

### 3.3 交互检测模块
- **职责**：检测玩家触摸操作类型（点击/滑动/拖拽），判断交互有效性。
- **与其他模块关系**：接收玩家输入，将有效交互传递给动作触发模块。
- **条件分支**：
  - if 触摸时间 < 0.5秒 → 判定为点击
  - if 触摸时间 >= 0.5秒 且 触摸时间 < 1秒 → 判定为滑动
  - if 触摸时间 >= 1秒 → 判定为拖拽
  - if 点击位置在角色碰撞区域内 → 有效点击
  - if 点击位置不在角色碰撞区域内 → 无效点击
  - if 滑动起点和终点都在角色碰撞区域内 → 有效滑动
  - if 滑动起点或终点不在角色碰撞区域内 → 无效滑动
  - if 拖拽终点在家具交互区域内 → 触发家具互动
  - if 拖拽终点不在任何家具交互区域内 → 角色回到原位

### 3.4 动作触发模块
- **职责**：根据交互类型和部位，播放对应动作动画和语音。
- **与其他模块关系**：从交互检测模块获取交互信息，调用动画和语音资源。
- **条件分支**：
  - if 交互类型 == 点击 → 根据点击部位播放对应动作
  - if 交互类型 == 滑动 → 根据滑动方向播放对应动作
  - if 交互类型 == 拖拽 → 根据拖拽终点家具播放对应动作
  - if 部位冷却时间未结束 → 播放轻推动作
  - if 部位冷却时间已结束 → 播放完整动作

### 3.5 小游戏模块
- **职责**：管理小游戏触发、进行、结算。
- **与其他模块关系**：从交互检测模块获取触发条件，结算后通知好感度模块。
- **条件分支**：
  - if 连续点击腹部3次（1.5秒内） → 触发挠痒小游戏
  - if 连续点击3个不同家具（6秒内） → 触发捉迷藏小游戏
  - if 小游戏进行中 → 锁定其他交互操作
  - if 小游戏胜利 → 播放胜利动画，增加好感度
  - if 小游戏失败 → 播放失败动画，不增加好感度

### 3.6 好感度模块
- **职责**：管理好感度数值，处理增加逻辑和每日上限。
- **与其他模块关系**：从动作触发模块和小游戏模块接收好感度增加请求。
- **条件分支**：
  - if 当日好感度增加量 < 30 → 允许增加好感度
  - if 当日好感度增加量 >= 30 → 拒绝增加，提示已达上限
  - if 好感度达到特定阈值（50/100/200） → 解锁新交互动作
  - if 好感度达到特定阈值（100/200/500） → 解锁新语音

### 3.7 成就模块
- **职责**：记录交互统计数据，检查成就解锁条件。
- **与其他模块关系**：从交互检测模块和小游戏模块获取统计数据。
- **条件分支**：
  - if 累计点击次数 >= 100 → 解锁“点击达人”成就
  - if 累计滑动次数 >= 50 → 解锁“滑动高手”成就
  - if 累计拖拽次数 >= 30 → 解锁“拖拽狂魔”成就
  - if 累计小游戏胜利次数 >= 20 → 解锁“小游戏冠军”成就
  - if 每个家具拖拽次数 >= 5 → 解锁“家具大师”成就

## 6. 系统流程

### 6.1 宿舍购买流程

1. 玩家在角色详情页点击“宿舍”按钮。
2. 系统检查玩家等级是否 >= 10。
   - 若等级不足，弹出提示“需要达到10级才能购买宿舍”，流程结束。
3. 系统检查玩家是否已拥有该角色。
   - 若未拥有，弹出提示“请先获取该角色”，流程结束。
4. 系统显示购买界面，包含宿舍预览、价格、购买按钮。
5. 玩家点击“购买”按钮。
6. 系统检查玩家货币是否足够。
   - 若货币不足，弹出提示“货币不足，请充值”，流程结束。
7. 系统扣除货币，将 dormitory_id 对应的 is_purchased 字段设为 true。
8. 系统弹出购买成功提示，解锁宿舍入口。
9. 流程结束。

### 6.2 进入宿舍流程

1. 玩家在宿舍列表中选择已解锁的宿舍。
2. 系统检查当前是否有其他宿舍正在加载。
   - 若有，弹出提示“请先退出当前宿舍”，流程结束。
3. 系统开始加载宿舍场景、角色模型、家具布局。
4. 加载过程中显示加载进度条。
5. 若加载超时（>5秒），显示加载失败提示，提供重试按钮。
   - 玩家点击重试，返回步骤3。
   - 玩家点击取消，返回宿舍列表。
6. 加载完成，角色出现在预设位置（如床边或沙发旁）。
7. 系统进入交互模式，等待玩家操作。
8. 流程结束。

### 6.3 点击交互流程

1. 玩家在宿舍场景中触摸屏幕。
2. 交互检测模块记录触摸位置和时间。
3. 系统判断触摸时间 < 0.5秒，判定为点击。
4. 系统检测点击位置是否在角色碰撞区域内。
   - 若不在，判定为无效点击，无反馈，流程结束。
5. 系统检测点击部位（头部、手臂、腹部、腿部等）。
6. 系统检查该部位的冷却时间是否已结束。
   - 若冷却未结束，播放轻推动作，流程结束。
7. 系统播放对应部位的动作动画 + 语音台词。
8. 系统记录该部位的最后交互时间，重置冷却计时器。
9. 系统通知好感度模块增加好感度+1。
10. 好感度模块检查当日好感度是否已达上限。
    - 若未达上限，增加好感度。
    - 若已达上限，不增加好感度，但交互动作正常播放。
11. 流程结束。

### 6.4 滑动交互流程

1. 玩家在角色身体上按住并滑动。
2. 交互检测模块记录触摸起点、终点和时间。
3. 系统判断触摸时间 >= 0.5秒 且 < 1秒，判定为滑动。
4. 系统检测滑动起点和终点是否都在角色碰撞区域内。
   - 若起点或终点不在角色上，判定为无效滑动，无反馈，流程结束。
5. 系统计算滑动距离是否超过阈值（50像素）。
   - 若不足，判定为无效滑动，无反馈，流程结束。
6. 系统根据滑动方向（上/下/左/右）播放对应动作动画 + 语音反馈。
7. 系统通知好感度模块增加好感度+1。
8. 好感度模块检查当日好感度上限。
9. 流程结束。

### 6.5 拖拽交互流程

1. 玩家长按角色（超过1秒）。
2. 交互检测模块检测到触摸时间 >= 1秒，判定为拖拽。
3. 角色进入拖拽模式，跟随玩家手指移动。
4. 玩家拖动角色到某个家具上方。
5. 系统检测角色当前位置是否落在家具交互区域内。
   - 若不在任何家具交互区域，角色保持跟随状态。
6. 玩家松开手指。
7. 系统判断松开时角色是否在家具交互区域内。
   - 若在，播放家具专属互动动作（如丢到床上、按到沙发上）。
   - 若不在，角色回到原位，播放轻微抗议动作。
8. 系统记录拖拽到该家具的次数（drag_to_count + 1）。
9. 系统通知好感度模块增加好感度+1。
10. 好感度模块检查当日好感度上限。
11. 流程结束。

### 6.6 挠痒小游戏流程

1. 玩家在角色腹部区域快速连续点击3次。
2. 交互检测模块记录每次点击的时间戳和位置。
3. 系统检查3次点击是否在1.5秒内完成。
   - 若超时，判定为普通点击，流程结束。
4. 系统检查所有点击是否落在腹部碰撞区域内。
   - 若有不在腹部的点击，判定为普通点击，流程结束。
5. 小游戏模块激活，进入挠痒小游戏。
6. 系统锁定其他交互操作，显示小游戏界面（计时器、得分）。
7. 角色开始随机移动躲避。
8. 玩家需要在10秒内点击角色身体任意部位累计10次。
9. 每次成功点击，得分+1，播放轻微反应动画。
10. 每次点击失败（点击空白处），不扣分，无反馈。
11. 时间到达10秒或得分达到10次时，小游戏结束。
12. 系统判断是否胜利（得分 >= 10）。
    - 若胜利，播放角色求饶动画 + 特殊语音，好感度+5。
    - 若失败，播放角色得意动画 + 普通语音，好感度不增加。
13. 系统解锁其他交互操作。
14. 流程结束。

### 6.7 捉迷藏小游戏流程

1. 玩家在宿舍中连续点击3个不同家具。
2. 交互检测模块记录每次点击的家具ID和时间戳。
3. 系统检查3次点击是否在6秒内完成。
   - 若超时，判定为普通点击，流程结束。
4. 系统检查3次点击的家具是否各不相同。
   - 若有重复，判定为普通点击，流程结束。
5. 小游戏模块激活，进入捉迷藏小游戏。
6. 系统锁定其他交互操作，显示小游戏界面（计时器、提示）。
7. 角色随机躲到5个可躲藏位置中的一个。
8. 系统显示提示文字“角色躲起来了，快找到她！”
9. 玩家需要在15秒内点击正确的家具找到角色。
10. 每次点击错误家具，剩余时间减少2秒，播放错误提示。
11. 点击正确家具时，小游戏结束。
12. 系统判断是否胜利（在时间内找到角色）。
    - 若胜利，播放角色惊喜动画 + 特殊语音，好感度+3。
    - 若失败（时间耗尽或点错3次），角色自己出来，播放无聊动画，好感度不增加。
13. 系统解锁其他交互操作。
14. 流程结束。
