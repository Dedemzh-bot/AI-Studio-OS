# 角色私人宿舍 - 宏观设计草案

## 一、系统概述与设计愿景

本系统为买断制角色私人宿舍，核心定位为 **纯展示与互动空间**。玩家通过一次性付费解锁特定角色的专属宿舍场景，在其中自由摆放角色、触发特殊互动动画（如触摸反馈、换装展示），无养成或资源产出。设计愿景是：为“极度媚宅的角色表现”提供私密、沉浸的展示舞台，强化玩家与角色的情感羁绊，驱动付费意愿。

## 二、核心规则与玩法机制

1. **买断解锁**：每个角色宿舍独立售卖，玩家付费后永久解锁该角色的宿舍场景及基础互动内容。
2. **自由摆放**：玩家可在宿舍场景中自由调整角色的位置、朝向、姿势（预设动作库），并切换不同服装（需拥有对应皮肤）。
3. **互动触发**：通过点击/拖拽角色特定部位（如头部、手部）触发预设的反馈动画、语音及表情变化。互动内容分级明确，严格控制在擦边暗示范围内。
4. **场景自定义**：提供少量可更换的家具/装饰品（如床、沙发、灯光色调），通过完成角色好感度任务或活动获取，不影响核心互动。
5. **无养成循环**：系统不产出任何资源或数值成长，纯粹为情感体验付费。

## 三、表现层与角色展示联动 (核心强制项)

1. **最大化角色展示**：
   - 默认摄像机视角为角色半身/全身特写，支持玩家手动旋转、缩放。
   - UI 元素极简化，仅在互动时显示提示图标，不遮挡角色主体。
   - 场景光照与材质渲染针对角色模型优化，突出皮肤质感、服装细节及物理效果（如头发飘动、软体碰撞）。
2. **特殊机位与擦边互动**：
   - 预设多个“心动机位”（如俯视、侧躺视角），玩家可一键切换。
   - 触摸反馈包含脸红、娇嗔语音、身体轻微扭动等，尺度控制在“暧昧暗示”级别。
   - 换装展示时，角色会做出展示服装的专属动作（如转圈、撩发）。
3. **严守擦边暗示底线**：所有互动动画与语音均不涉及直接性行为或露点，符合“灰色地带”分级要求。

## 四、经济循环与商业化埋点

1. **直接付费**：角色宿舍作为独立商品，定价为中等价位（如 30-60 元人民币/个），或包含在角色礼包中。
2. **间接付费驱动**：
   - 宿舍内可展示的服装需通过抽卡或商城购买获得，刺激皮肤消费。
   - 部分高级互动动画（如特殊触摸反馈）可捆绑限定皮肤出售。
3. **无资源产出**：系统不产出抽卡资源或养成材料，避免影响核心经济循环。

## 五、旧系统与数据联动 (强依赖)

1. **角色系统**：宿舍解锁与角色拥有状态绑定，仅可解锁已拥有的角色。
2. **服装系统**：宿舍内可切换的服装列表同步自玩家已拥有的皮肤数据。
3. **好感度系统**：宿舍互动可触发好感度语音，但好感度数值仍由外部系统（如战斗、剧情）控制，宿舍仅作为展示窗口。
4. **大厅系统**：宿舍可作为独立入口，与主大厅并列，玩家可快速切换。

## 六、待确认风险与追问

1. **技术性能**：高精度角色模型与场景同时渲染对移动端性能的冲击，需评估是否支持低配机型。
2. **内容生产量**：每个角色需独立制作场景、互动动画及语音，人力成本较高，需确认首发角色数量及后续更新节奏。
3. **定价策略**：买断制定价需平衡玩家付费意愿与内容价值，是否考虑推出“宿舍通行证”打包多个角色。
