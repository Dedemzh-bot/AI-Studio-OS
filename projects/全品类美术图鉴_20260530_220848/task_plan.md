### 2. 执行顺序与依赖

```mermaid
graph TD
    subgraph 阶段一：设计细化与数据定义
        SP-01[SP-01: 系统详细设计]
        ST-01[ST-01: 数据Schema定义] --> SP-01
    end

    subgraph 阶段二：并行设计与数值
        UI-01[UI-01: UI/UX设计] --> SP-01
        NP-01[NP-01: 数值与奖励设计] --> SP-01
        CA-01[CA-01: 战斗数据联动] --> SP-01
    end

    subgraph 阶段三：开发与审查
        CA-01_Code[CA-01: 核心逻辑实现] --> ST-01
        CA-01_Code --> NP-01
        CA-01_Code --> CA-01
        
        CA-02_Code[CA-02: 前端展示实现] --> UI-01
        CA-02_Code --> ST-01

        AA-01[AA-01: 数值平衡性审查] --> NP-01
    end

    subgraph 阶段四：集成与测试
        Integration[集成测试] --> CA-01_Code
        Integration --> CA-02_Code
        Integration --> AA-01
    end

    SP-01 --> |串行依赖| 阶段二
    阶段二 --> |并行| 阶段三
    阶段三 --> |串行| 阶段四
```

- **串行依赖**：
    - `SP-01` (系统详细设计) 是所有下游任务的**前置条件**，必须最先完成。
    - `ST-01` (数据 Schema 定义) 必须在 `SP-01` 之后，但在 `CA-01_Code` 和 `CA-02_Code` 之前。
    - `AA-01` (数值审查) 必须在 `NP-01` (数值设计) 之后，且最好在 `CA-01_Code` 开始实现奖励发放逻辑之前完成，以避免返工。

- **并行任务**：
    - `UI-01`、`NP-01`、`CA-01` 可以**并行**进行，因为它们都依赖于 `SP-01`，但彼此之间没有直接依赖。
    - `CA-01_Code` (核心逻辑) 和 `CA-02_Code` (前端展示) 可以**并行**开发，因为它们分别依赖于 `ST-01`/`NP-01`/`CA-01` 和 `UI-01`/`ST-01`。

### 3. 风险提示

- **阻塞点**：
    - **`SP-01` 延期**：这是整个项目的关键路径，任何延迟都会导致所有下游任务延期。
    - **`ST-01` 定义不清**：如果数据 Schema 定义不完整或与 UI/后端需求冲突，会导致 `CA-01_Code` 和 `CA-02_Code` 大量返工。
    - **美术/音频产能不足**：设计草案中已指出此风险。如果首批上线内容的美术资产（模型、立绘、CG、音乐）无法按时提供，图鉴系统将无法上线。**需要与美术/音频团队提前对齐资源交付时间表**。
    - **“已绝版”条目策略未定义**：草案中已提出此风险。如果未在 `SP-01` 中明确“已绝版”条目的展示策略，会导致 UI 设计和后端逻辑无法闭环。

- **跨团队依赖冲突**：
    - **与战斗/关卡团队**：`CA-01` 需要战斗/关卡团队提供敌人/场景的 ID 列表和解锁事件触发点。如果对方团队接口变更，需要及时同步。
    - **与剧情/音频团队**：CG 和音乐的解锁事件需要与剧情/音频团队的数据结构对齐。
    - **与活动团队**：活动限定内容的图鉴条目需要与活动团队的数据结构对齐，并确保活动结束后条目永久保留的逻辑正确。
    - **与美术团队**：UI Agent 的设计稿需要与美术团队的美术风格规范对齐，确保图鉴界面与游戏整体视觉风格一致。
